# 경영 확장 설계 — "벽의 양쪽을 다 겪게" (2막 단막극)

> **한 줄 요약**
> 플레이어가 지금은 벽에 *부딪히는* 응급의 의사만 하지만, 이 확장은 벽을 *세우는* 병원(전화를 받는 쪽)까지 플레이하게 해서, "합리적 공범"이 되는 경험을 다크코미디로 겪게 한다. 같은 판정 로직을 1막에선 내가 휘두르고 2막에선 그 로직에 내가 부딪힌다.

## 1. 확정된 설계 결정 (브레인스토밍 산물)

다섯 개의 큰 갈림길을 사용자와 하나씩 넘겨 잠갔다:

1. **톤 = 합리적 공범(A)을 다크코미디로.** 재미의 정체는 "병원 키우는 손맛"도 "공범이 돼서 통쾌함"도 아니고, **불편한 사실을 정면으로 마주할 때 나오는 웃픈 재미**(다크코미디). 레퍼런스: 《Papers, Please》의 관료적 태연함 + "흑자 달성 🎉"을 명랑하게 축하하는 경영 UI와 조용히 비어가는 순환기 배후 사이의 **낙차**.
2. **구조 = 2막 단막극(구조 1)**, 나중에 다회차 캠페인(구조 2)으로 성장. 둘은 대립이 아니라 같은 축의 "지금/나중".
3. **경제 깊이 = 사전확정 + 판정(A).** 투자 결정은 위저드에서 **1회**(과·의사 선택 = 투자). 재투자 루프는 구조 2.
4. **철학 = agency가 결과에 닿는다(ii).** 1막에서 순환기를 지었느냐가 2막에서 환자 생사를 가른다. 지금 게임의 "무력함" 논지를 *"인센티브가 죽음을 합리로 만든다"*로 진화(장부 에필로그 PR #14가 이미 그 문을 엶).
5. **양쪽 다 값을 치른다.** 공범(순환기 스킵) → 비극(전원 실패) + 흑자 장부. 양심(순환기 건설) → 생존 + 적자·소송으로 피 흘리는 장부. 어느 값을 치를지가 딜레마.

### 톤 가드레일 (다크코미디가 선정성·조롱으로 안 미끄러지게)
- 농담의 과녁은 **언제나 시스템의 태연한 논리**(=합리적 선택이 곧 사람 죽는 선택이라는 부조리). **죽는 환자·특정 집단(의사/정부/환자)은 절대 과녁이 아니다.**
- 코미디는 "명랑한 경영 대사"와 "인간의 대가" 사이의 **극적 아이러니**에서만 나온다. 고통을 희화화하면 게임이 무너진다.
- **톤의 분업**: 다크코미디는 **1막에만**. **2막·결말은 냉정·비극**(기존 영수증/장부의 "해석 0, 라벨+숫자" 원칙 유지, PR #13/#14). 명랑함이 세워둔 낙차를, 냉정함이 떨어뜨린다. 결말은 농담하지 않는다.
- 기존 game-concept.md §6 가드레일과 동일선상: 특정 인물·집단 비난 금지, 과장·선정성 경계, 교육·공감이지 선동 아님.

## 2. 한 판의 전체 흐름

```
[1막] 받는 병원 (다크코미디 · 경영)
  설정 위저드  →  콜 큐(수용/거절 판정)  →  명랑한 장부 갱신
        │
   막간: "이번 분기 흑자 +XXX억 🎉"  …  그런데 응급이 뜬다 → 시점 전환
        │
[2막] 응급 (내 병원에서 출발) ── 1막 선택으로 분기
  ├─ 순환기 배후 지었으면 → 내 응급실이 직접 PCI → 산다 (장부는 적자·소송으로 피 흘림)
  └─ 순환기 배후 스킵했으면 → 밖으로 전원 = 기존 STEMI 뺑뺑이 → 빡빡한 벽, 대개 놓침
        │
[결말] 전원 영수증(debrief) + 병원 장부(ledger) — 이제 "내 병원"의 데이터
```

세션은 5개 페이즈의 상태기계: `SETUP → RECEIVING → INTERSTITIAL → EMERGENCY → EPILOGUE`.

## 3. 1막 — 받는 병원 (다크코미디 · 경영)

### 3.1 설정 위저드 (다크코미디 첫 비트)
- **입력**: 병원 이름(자유 텍스트), 둘 과 선택 + 과별 의사 수, 예산 한도 내.
- **명랑한 유도 카피**(부호는 근거 준수, §7): `미용·피부 — 수익 예상 ↑`, `건강검진 — 흑자`, `순환기내과(PCI) — 적자 예상 · 소송 리스크 ⚠`, `흉부외과/산부인과 — 적자 · 소송 ⚠`.
- **선택 → 제약**: 위저드 산출물이 곧 플레이어 병원의 `Hospital` 객체 + `HospitalEconomics`. 핵심 축은 **순환기내과** — 의사 ≥1로 두면 `backupCare`에 `CARDIOLOGY` 포함, 아니면 2막에서 STEMI를 **물리적으로 못 받는다**.
- **이번 주 단순화**: 병상은 **소량 고정값**(위저드에서 안 고름). **병동 증설·장비 업그레이드는 구조 2**("나중에").

### 3.2 콜 큐 (Papers, Please의 심장)
콜이 하나씩 들어오고, **기존 `adjudicateTransfer(playerHospital, patient)`를 플레이어 손으로** 돌린다. 세 종류의 콜:

| 콜 종류 | 순환기 없는 병원 | 순환기 있는 병원 | 코미디 포인트 |
|---|---|---|---|
| **STEMI형**(순환기 배후 필요) | 하드락 거절(못 받음) — 벽을 안쪽에서 배움 | **적자·소송 무릅쓰고 받을까** 선택 | 필수 케이스에 시스템이 얼마나 쉽게 "안 됩니다"를 말하는지 |
| **일반 응급**(병상만 필요) | 병상 있으면 받을 순 있으나 저마진·법적 노출 vs 병상을 비급여에 남길까 → 선택 | 동일 선택 | 병상 하나의 기회비용 |
| **미용·검진 워크인** | `받으시겠습니까? 물론이죠 🎉` | 동일 | 시스템이 얼마나 **신나게** "예"라고 하는지 |

- **장부는 그 옆에서 명랑하게 갱신**된다 — 거절/비급여 수용마다 순이익 ↑, "이번 분기 흑자 🎉".
- **핵심**: 벽은 *강요된* 거절이 아니라 플레이어가 *고른* 거절로 쌓인다. 그게 공범의 정체. (순환기를 아예 스킵한 병원은 STEMI 콜이 전부 하드락 거절 → 콜 행렬이 "받을 수 없음"의 퍼레이드 + 치솟는 흑자. 이 자체가 서늘한 코미디 비트.)

### 3.3 법적 리스크 (이번 주엔 가볍게)
- 위험한 필수 케이스를 수용하면 **소송 노출**이 누적, 결말 장부에 **소송 비용 한 줄**로 실현(결정론, RNG 없음 — 예: 특정 임계 이상 위험 수용 시 확정 비용).
- "필수과 = 적자 + 소송"의 두 번째 이빨. 리치한 소송 이벤트 시스템은 구조 2.

## 4. 막간 — 시점 전환
"분기 마감. 순이익 +XXX억 🎉"(명랑) → 톤이 꺾이며 응급이 뜬다. **이번엔 당신 응급실로 STEMI가 온다.** 가능하면 1막에서 거절/못 받은 케이스와의 연결을 카피로 암시(선택적).

## 5. 2막 — 응급 (1막 선택으로 분기)

플레이어는 이제 **1막에서 지은 자기 병원의 응급의학과 의사**. STEMI 환자가 도착.

- **순환기 배후 지었으면**: 내 응급실이 직접 PCI → **생존.** 대신 1막 장부가 적자·소송으로 피 흘린다. (딜레마의 다른 값 — 전원 미니게임 없이 짧은 생존 비트)
- **순환기 배후 스킵했으면**: 내가 시술 못 하니 **밖으로 전원 = 기존 STEMI 뺑뺑이 그대로**(`createStemiScenario`의 외부 병원 풀 재사용). 기존 시나리오는 "풀 수 있되 빡빡하게" 설계돼 있으니 **rigged 사망이 아니라 같은 정직한 벽**으로 — 대개 놓치지만 가능성은 열려 있다. 그리고 벽을 세운 게 나라는 걸 안다.

이 분기가 철학 (ii)를 구현한다: 지으면 확정 생존·적자, 스킵하면 빡빡한 벽에 던져짐. **기존 게임을 안 버리고 감싼다.**

> ⚠️ 밸런싱 노트: 기존 시나리오의 "풀 수 있되 빡빡" 성질(출구 h6 존재)은 의도적으로 보존한다. 공범을 *flat하게 불가능*으로 만들면 플레이어가 배신감을 느낀다("반드시 풀 수 있게 설계" 원칙, U2). 공범의 복잡성은 "왜 내가 이 뺑뺑이에 있나 = 내가 순환기를 안 지어서"에서 온다.

## 6. 결말 — 영수증 + 장부, 이제 "내 것"

기존 `debrief`(전원 영수증) + `ledger`(병원 장부)를 **플레이어 병원 데이터로** 재사용. 지금은 가상의 한바다대학병원이 하드코딩돼 있지만, 이제 **플레이어가 지은 병원의 순이익·채용**에서 파생. 세 분기:

| 분기 | 화면 | 낙차 |
|---|---|---|
| **공범 · 사망** | 전원 영수증(사유별 집계) + 내 장부(순이익 +XXX억, 순환기 채용 **0**) | 못 보낸 환자 옆에 내가 세운 흑자 장부 |
| **공범 · 생존**(운 좋게 전원) | 아슬한 전원 영수증 + 내 장부(흑자, 순환기 0) | "이번엔 남이 그 역량을 대신 짊어졌다" |
| **양심 · 생존** | 전원 미니게임 없음 + 내 장부(적자 + 소송 비용 + 순환기 채용 N) | "환자는 살았다. 그 값이 여기 있다" |

- **채용 0/N은 하드코딩이 아니라 위저드 선택에서 파생**(`essentialHires` = 순환기 의사 수). 기존 `ledger.ts` 원칙 유지.
- 결말 카피는 **라벨+숫자, 해석 최소**(PR #13/#14). 다크코미디는 1막에서 이미 낙차를 세웠고, 결말은 그걸 냉정하게 떨어뜨릴 뿐.

## 8. 아키텍처 — 기존 패턴 그대로 (순수함수 + TDD)

### 8.1 신규 순수 모듈 (`src/game/`)
- **`setup.ts`** — 과 카탈로그(수익/필수, 채용비, 분기 손익 부호, 소송 플래그, 제공 배후과) + `buildHospital(choices) → { hospital: Hospital, economics: HospitalEconomics }`. 순수·결정론.
- **`receiving.ts`** — 콜 큐 타입(`IncomingCall`), `classifyCall(hospital, call) → 'HARDLOCK_REJECT' | 'CHOICE'`(기존 `adjudicateTransfer` 재사용), `decide(state, call, accept) → ReceivingState`(순수 리듀서, 장부·소송 노출 누적, 하드락 콜 수용 시도 가드).
- **`session.ts`** — 2막 세션 상태기계: `SessionPhase`, `SessionState`, `completeSetup / completeReceiving / beginEmergency(분기) / toEpilogue`. `beginEmergency`가 `hospital.backupCare` 유무로 in-house 생존 vs 기존 `startGame(...)` 전원 라운드로 분기.

### 8.2 재사용 (변경 없음 또는 최소)
- `adjudicate.ts`(양쪽에서 같은 로직) · `goldenTime.ts` · `round.ts`(2막 전원) · `debrief.ts` · `ledger.ts` · `scenarios.ts`(2막 외부 풀).
- **타입 변경(types.ts)**: `Department` 카탈로그 개념 추가(수익과 `AESTHETICS`/`CHECKUP` + 필수과는 기존 `Specialty` 매핑), `SetupChoices`, `IncomingCall`. 기존 `Hospital`/`HospitalEconomics`/`Patient`/`RejectionReason`는 그대로 소비.

### 8.3 UI (`src/components/`)
- 신규: `SetupWizard`, `ReceivingPhase`(콜 큐 + 명랑 장부).
- 재사용: 기존 응급/전원 UI(`GameClient`의 2막 부분), 영수증·장부 결말 패널.
- 대사: 1막 다크코미디 **폴백 대사**를 `dialogue.ts`에 추가(결정론). 실LLM은 다음 주 X8에서 이 위에 얹음.

```
toEpilogue ─▶ debrief(GameState) + ledger(HospitalEconomics)  ← 내 병원 데이터
```

## 9. 테스트 전략 (TDD · Red→Green→Refactor)

- **setup**: 합리적 빌드(필수과 0) → economics 흑자·`essentialHires 0`·`backupCare`에 CARDIOLOGY 없음. 양심적 빌드 → `essentialHires>0`·CARDIOLOGY 포함·segments 적자. 병상/필드 파생 정확.
- **receiving**: 하드락 콜 → `decide(accept=true)` 거부/가드. 선택 콜 → 수용·거절 둘 다 허용. 장부·소송 노출 갱신. 미용 워크인 → 항상 수용 가능.
- **session**: 페이즈 전이(SETUP→…→EPILOGUE), `beginEmergency` 분기(backupCare 유무 → in-house vs 전원), 종료 후 가드.
- **통합 불변식**: 공범 병원(CARDIOLOGY 없음) → 2막이 전원 `GameState`(IN_PROGRESS, 외부 풀). 양심 병원 → in-house 생존. **순환기 없으면 STEMI 하드락**이 받는 쪽/보내는 쪽 양쪽에서 성립.
- **기존 불변식 보존**: 병상0 하드락, 설득으로 안 뚫림 등 회귀.
- 게이트: 전 테스트 green · `tsc --noEmit` · `next build`(Turbopack) · 브라우저 관찰 검증(공범·양심 양 경로 완주).

## 12. 성공 기준
- API 없이 **한 세션 완주**: 병원 설립 → 콜 수용/거절(명랑 장부) → 시점 전환 → 2막 분기 → 내 병원 장부로 결말.
- **공범 경로**(순환기 스킵 → 전원 뺑뺑이 → 흑자 장부 + 채용 0)와 **양심 경로**(순환기 건설 → 생존 → 적자·소송 장부) **둘 다 도달 가능**.
- 기존 불변식·기존 8/10 제출 게임이 깨지지 않음(비파괴 감싸기).
- 전 테스트 green · `tsc` · `next build` 통과 · 브라우저 검증 · 다크코미디 톤이 1막에, 냉정·비극이 2막·결말에.
