# T-067 · 자를 안 재고 그 자로 화면을 쟀다

- **증상**: 의무기록 테마를 전 화면에 적용하며 WCAG 대비를 브라우저에서 전수 측정했다. 감사 스크립트가 **43건 실패**를 뱉었다 — 본문·금액·장부가 1.0~2.5:1이라는 참담한 숫자였다. 고치려고 보니 값이 이상했다: `text-ink`(#26221a)가 마닐라 종이 위에서 **1.2:1**로 나왔는데, 그 조합은 스펙이 AAA 11.59로 감사해 둔 자리다. 스크립트를 세 번 고쳤고 **세 번 다 새로운 거짓 실패 목록**이 나왔다.

- **원인**: 결함이 화면이 아니라 **자(측정 코드)**에 있었다. 셋 다 서로 다른 원인이다.

  1. **oklab을 RGB로 파싱** — Tailwind v4의 알파 유틸(`text-on-desk/70`)은 computed style이 `oklab(0.853 -0.003 0.043 / 0.7)`로 나온다. 정규식 `match(/[\d.]+/g)`가 이 숫자들을 R·G·B로 읽어 **0.85, -0.003, 0.043을 색으로** 계산했다 → 전부 1.0~1.2:1.
  2. **그라디언트 배경 = 투명** — `.paper-card`는 `background: linear-gradient(...)` 축약형이라 `background-color`가 `transparent`다. 조상을 거슬러 배경을 찾는 함수가 종이를 건너뛰고 **어두운 책상**을 배경으로 잡아, 종이 위 잉크를 전부 dark-on-dark로 판정했다.
  3. **슬래시 알파를 불투명으로 오인** — 반투명 판정을 `/,\s*[\d.]+\)$/`(쉼표 뒤 알파)로 했는데 oklab은 `... / 0.6)`처럼 **쉼표가 없다**. 반투명 배경에서 조상 합성을 멈춰, 종이 위 셀을 책상 위 셀로 쳤다.

  공통점: 세 번 다 **에러가 없었다**. 스크립트는 완주하고 정렬된 실패 목록을 냈고, 숫자에 소수점이 붙어 있어 측정처럼 보였다. "43건 실패"는 화면에 대한 사실이 아니라 **자에 대한 사실**이었다.

- **해결**: 색 파싱을 직접 하지 않는다 — **브라우저에 시킨다.** 1×1 캔버스에 배경을 칠하고 그 위에 전경을 칠한 뒤 픽셀을 읽으면, 색 공간(oklab·lab·color())·알파·합성을 전부 브라우저가 처리한 결과가 sRGB로 나온다. 불투명 판정도 정규식이 아니라 **칠해 보고 알파 채널이 255인지** 본다. 배경은 조상 체인을 모아 **불투명한 쪽부터 아래로 순서대로 합성**한다(`.paper-card`는 그라디언트라 어두운 끝 `--paper-2`를 최악값으로 쓴다).

  ```js
  const px = () => { const d = ctx.getImageData(0,0,1,1).data; return [d[0],d[1],d[2]]; }
  const isOpaque = c => { ctx.clearRect(0,0,1,1); ctx.fillStyle = c; ctx.fillRect(0,0,1,1)
                          return ctx.getImageData(0,0,1,1).data[3] === 255 }
  ```

  고친 자로 재니 실패가 **43건 → 0건**으로 떨어졌다(콜 접수 52개 텍스트, 최저 4.56). 그리고 **그 43건 중 진짜가 있었다**: ① `on-desk-muted`가 desk-2 위 3.82:1(AA 미달)인데 내가 본문까지 그 토큰으로 칠했다 ② 맵 방 라벨을 토큰화하며 5.4 → 2.37로 떨어뜨렸다 ③ 달력 흑자 칸에 배경 틴트를 깔아 초록 잉크가 4.10까지 내려갔다. 진짜 셋은 거짓 40건에 묻혀 있었다.

- **재발방지**: **자를 먼저 검증한다 — 이미 답을 아는 입력으로.** 스펙에 감사값이 적힌 조합(`ink/paper = 11.59`, `go/paper-2 = 4.55`)이 있었고, 그 하나만 스크립트에 먼저 넣어 봤으면 1.2가 나오는 순간 자가 틀렸음을 알았을 것이다. 화면을 재기 전에 **자로 자를 잰다**: 알려진 기준값이 재현되지 않으면 그 뒤 숫자는 전부 무의미하다.

  더 일반적으로 — **파싱은 재구현하지 말고 런타임에 위임한다.** 색·날짜·URL처럼 스펙이 계속 자라는 것을 정규식으로 뜯으면, 새 표기가 등장한 날 조용히 틀린다(oklab은 Tailwind v4가 알파 유틸을 oklab으로 바꾸면서 들어왔다 — 내 정규식은 v3 시절 가정이었다). 브라우저는 그 스펙의 정본 구현을 이미 갖고 있다.
