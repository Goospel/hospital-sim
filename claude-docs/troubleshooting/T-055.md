# T-055 · 게이트와 실행이 특수케이스를 비대칭 처리하면 판정과 자원이 어긋난다

- **증상**: 일반응급(`GENERAL_EMERGENCY`, 설계 A — "응급실 상주 인력이 받으므로 배후 전문의 점유 없이 자유 전문의 유무와 무관하게 수용")을 받았는데, **같은 날 뒤따르는 콜이 이유 없이 벽(`NO_FREE_SPECIALIST`)에 막혔다.** 자유 외과의가 하나 사라진 것처럼 행동한다 — "받을 수 있다"고 판정된 콜이 실제로는 전문의 자원을 소모하고 있었다.
- **원인**: 자원 게이트와 실행이 **다른 함수로 나뉘어** 있고, 특수케이스 제외를 **한 쪽에만** 넣었다. `hardlockReason`(게이트: 이 콜을 받을 수 있나?)은 `GENERAL_EMERGENCY`를 점유 벽에서 제외해 "전문의를 안 쓴다"고 판정했지만, `decide`(실행: 실제 배정)의 점유 가드는 그 예외를 빠뜨려(`if (effectiveAccept && free.length > 0)`) 자유 전문의를 점유해버렸다. 두 함수가 같은 케이스를 **비대칭으로** 봐서 판정(“안 씀”)과 실제 상태(“씀”)가 어긋났다. tsc·vitest는 둘 다 타입·단위상 정상이라 안 잡는다 — 상태 불일치는 시뮬레이션을 굴려봐야 드러난다.
- **해결**: `decide`의 점유 가드에도 동일 예외를 추가해 게이트와 대칭으로 맞췄다 — `if (effectiveAccept && free.length > 0 && call.kind !== 'GENERAL_EMERGENCY')` ([src/game/receiving.ts:377](../../src/game/receiving.ts), 94c9767). 더 나은 형태는 "이 콜이 전문의를 점유하는가"를 **단일 술어**(예: `occupiesSpecialist(call)`)로 뽑아 게이트·실행이 **같은 출처**를 읽게 하는 것 — 그러면 대칭이 구조적으로 강제된다(이중 기재 제거와 같은 발상).
- **재발방지**: "받을 수 있나?"를 판정하는 **게이트**와 "실제로 자원을 소모하는" **실행**이 나뉘어 있으면, 특수케이스 제외는 **양쪽에 동시에** 넣거나 단일 술어에서 파생시켜라. 리뷰 대칭 체크 한 줄: *"게이트가 예외로 빼는 케이스를 실행도 같은 예외로 처리하나?"* — 이 트랩은 **SDD 리뷰어가 잡았고 코디네이터(나)가 등재를 흘렸다.** 그 누락(T-054 이후 8 PR·트러블슈팅 0개)이 CLAUDE.md 「SDD 종료 trap 스윕」 신설의 계기다 — 서브에이전트가 디버깅한 함정은 코디네이터가 회수해야 등재된다.
