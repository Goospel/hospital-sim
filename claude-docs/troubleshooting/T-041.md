# T-041 · 트레일러 앞에 **빈 줄**을 넣으면 `Skills-used`가 기계 집계에서 통째로 누락된다

- **증상**: `CLAUDE.md`의 AI 활용 추적 규칙대로 `Skills-used:` / `Plugins-used:` / `Skill-benefit:` 트레일러를 커밋 메시지 끝에 넣었는데, `git log -1 --format="%(trailers:only=true)"`가 **`Co-Authored-By` 한 줄만** 뱉었다. 눈으로 보면 메시지에 다 들어 있어서 정상으로 보인다 — **무성 실패.**
- **원인**: git은 **메시지의 마지막 문단(paragraph)만** 트레일러 블록으로 파싱한다. `Skill-benefit:` 줄들과 `Co-Authored-By:` 사이에 **빈 줄**을 넣어 문단을 둘로 쪼갠 게 원인. 앞 문단은 그냥 본문 텍스트가 되어 `--grep`·`interpret-trailers --parse` 집계에서 사라진다. `CLAUDE.md`의 예시는 빈 줄 없이 붙어 있는데 그걸 어겼다.
- **해결**: 트레일러를 **빈 줄 없이 한 덩어리로** 붙인다. `Skills-used` → `Plugins-used` → `Skill-benefit`(반복 가능) → `Co-Authored-By`가 **연속**이어야 한다.
- **재발방지**: 한글 커밋(T-026)으로 `git commit -F` 한 뒤 **`git log -1 --format="%(trailers:only=true,unfold=true)"`로 파싱 결과를 눈으로 확인**한다. 커밋 메시지 본문에 트레일러가 "보이는 것"은 파싱됐다는 증거가 아니다. ⚠️ 이 프로젝트는 **모든 커밋**에 트레일러를 요구하므로(스킬 미사용 시에도 `Skills-used: none`) 재발률이 사실상 100%다. 목적이 "빈도 카운트 · 미사용 스킬 diff"라는 **기계 집계**인 만큼, 파싱 안 되는 트레일러는 안 쓴 것과 같다.
- **→ 승격됨 (2026-07-17) · 검사기**: 위 예측("재발률이 사실상 100%")이 **그대로 적중했다.** 이 항목을 쓴 뒤 커밋 4개(#38·#39·#40·#41)가 **전부 파싱=0**으로 재발했고, 재발방지 문구("커밋 후 `%(trailers:only=true)`로 눈으로 확인한다")는 4번 다 실행되지 않았다. **함정을 알고, 문서로 남기고, 자기가 안 지켜질 것까지 예측하고, 정확히 그대로 재발한 사례** — 소프트 규칙으로는 이 계열이 안 잡힌다는 결정적 증거다. → **`.githooks/commit-msg`** 신설(`git config core.hooksPath .githooks`): `Skills-used`가 없거나 **트레일러로 파싱되지 않으면 커밋 자체를 거부**하고, `git`이 실제로 보고 있는 트레일러를 출력해 원인을 즉시 보여준다. 머지·revert·fixup·빈 메시지는 통과. ⚠️ 훅 파일의 EOL은 [T-046](T-046.md) 참조 — CRLF로 풀리면 검사기가 조용히 사라진다.
- **실측 (2026-07-17, 규칙 도입 `69494bd` 이후 15커밋)**: 준수율 **15/15 = 100%** · `--grep` 집계 **15/15 = 100%** · `interpret-trailers` 파싱 **2/15 = 13%** · **집계를 실제로 돌린 횟수 = 0**(그날이 처음). **규칙은 100% 지켜졌는데 그 규칙이 존재하는 이유는 0이었다.** 집계는 내내 가능했는데(그래서 `--grep`으로 즉시 뽑혔다) 아무도 돌리지 않았다 → 그래서 CLAUDE.md에 **"언제 집계하는가"**(④ 문서 갱신 시·P7 최종화 시)를 함께 못박았다. 소비자를 지정하지 않은 규약은 **지켜지면서도 죽는다.**
