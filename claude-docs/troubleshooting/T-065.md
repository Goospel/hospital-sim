# T-065 · compositing 없는 탭에선 CSS 애니메이션 timeline이 아예 안 흐른다

- **증상**: Task 5(연속 시간 흐름 브라우저 실측)에서 배경 보행자(`.hm-walker`)의 `getBoundingClientRect().x`를 1초 간격으로 여러 번 재 "결정 대기 중에도 배경이 움직인다"를 확인하려 했다. `setInterval`로 50ms마다 좌표를 20초 넘게 찍었는데 **단 한 번도 값이 바뀌지 않았다** — `x`가 소수점까지 완전히 동일했다. 이동 중인 의사 아바타(`transition-all duration-300`)도 같은 증상: `top` 스타일이 32.24%(ROOM)와 60%(CORRIDOR)로 서로 다른데 `getBoundingClientRect().top`은 두 아바타가 **똑같은 픽셀값**을 냈다 — 마치 둘 다 CORRIDOR에 있는 것처럼.
- **원인**: 이 환경은 스크린샷 compositing이 비활성화된 탭이라 `document.hidden === true`(`document.visibilityState === "hidden"`)이고, `requestAnimationFrame` 콜백이 **한 번도 실행되지 않았다**(3초 대기해도 로그 0건). `element.getAnimations()[0]`을 찍어보면 `playState: "running"`인데 `currentTime: 0`, `timeline.currentTime: 0`— 애니메이션은 "재생 중"으로 보고되지만 타임라인 자체가 0에 박제돼 있다. Chrome은 렌더링 파이프라인(스타일→레이아웃→페인트→합성)의 "update the rendering" 단계에서 document timeline을 전진시키는데, compositing이 요청되지 않는 탭은 이 단계가 스케줄되지 않아 **CSS 애니메이션·트랜지션 둘 다** 영구히 로컬 타임 0(=시작 직전) 상태로 렌더된다. 반면 `setTimeout`은 매크로태스크라 렌더링 파이프라인과 무관하게 정상 발화한다(`useHospitalClock`이 `setTimeout` 기반이라 시계 자체는 정확히 흘렀다) — 애니메이션/트랜지션만 죽고 상태 기반 로직은 멀쩡했던 게 혼란의 핵심.
- **해결**: 두 갈래로 우회했다. ⓐ **논리적 목표값 확인**: `el.style.left`/`el.style.top`(React가 커밋한 인라인 style 문자열)은 트랜지션 진행과 무관하게 항상 최신 타깃 값이다 — `getBoundingClientRect()` 대신 이걸 읽으면 "이동이 단조인가"를 렌더 없이도 검증할 수 있다. 더 근본적으로는 DOM 노드의 `__reactFiber$...` 키를 타고 올라가 `HospitalMap`/`ReceivingPhase`의 `memoizedProps`(`scene`, `receiving`)를 직접 읽어 게임 상태 자체(atMin·busyUntil·avatars)를 authoritative하게 확보했다. ⓑ **렌더링 관계 확인(겹침·paint order)**: `animation-play-state: paused` + **음수 `animation-delay`**를 인라인으로 주면 타임라인이 안 흘러도 "그 프레임"을 정적으로 그릴 수 있다(delay가 음수면 로컬 타임이 즉시 그만큼 양수로 시작). 이 스크럽으로 배경 보행자를 실제 아바타와 같은 x 좌표에 세우고 `document.elementFromPoint()`로 paint order(실제 아바타가 항상 위)를 확인했다.
- **재발방지**: compositing이 막힌(또는 백그라운드) 탭에서 애니메이션·트랜지션 기반 UI를 실측할 땐 **`getBoundingClientRect()`의 시계열 변화를 곧바로 "안 움직인다"로 읽지 않는다** — 먼저 `el.getAnimations()[0]?.timeline?.currentTime`이 실제로 전진하는지, `requestAnimationFrame`이 몇 초 안에 발화하는지부터 확인한다. 전진하지 않으면 렌더 경로가 아니라 ⓐ 상태 소스(React Fiber `memoizedProps`, 혹은 인라인 style 문자열)나 ⓑ `animation-delay` 음수 스크럽으로 우회한다. `setTimeout` 기반 로직(게임 시계 등)은 이 문제의 영향을 받지 않으므로 "시계는 도는데 그림만 안 움직인다"는 관찰 자체가 이 증상의 신호다. 계열: [T-064](T-064.md)가 "좌표가 공식과 맞는 걸 확인했지 대상과 맞는 걸 확인하지 않았다"였다면, 이번은 "관계를 확인하려 했는데 측정 도구 자체가 얼어 있었다" — 관계형 질문(T-064의 교훈)을 실제로 쓰려면 그 관계를 그리는 렌더링 파이프라인이 살아있는지부터 검증해야 한다는 한 겹 앞의 전제다.
