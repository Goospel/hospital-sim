# T-052 · .git/info/exclude로 가린 파일은 clone에 안 따라간다 — 기기 간 공유는 .gitignore + 추적

- **증상**: 데스크톱 Obsidian은 폴더 숨김이 적용돼 정리된 문서만 보이는데, 같은 repo를 클론한 랩탑 Obsidian은 `node_modules/`·`.next/`·`.claude/worktrees/` 등 모든 폴더가 파일 탐색기에 노출됐다. 폴더 숨김 설정(`.obsidian/app.json`의 `userIgnoreFilters`)이 랩탑엔 없음. `.gitignore`를 grep해도 `obsidian` 문자열이 안 나와 "왜 무시됐는지"조차 안 보였다.
- **원인**: `.obsidian`은 `.gitignore`가 아니라 **`.git/info/exclude`**(로컬 전용, 커밋·push·clone에 실리지 않음)로 무시되고 있었다. exclude는 그 기기의 `.git`에만 존재하므로 **`.obsidian`은 애초에 한 번도 커밋된 적이 없고**, 랩탑 clone엔 폴더 자체가 오지 않았다. git의 무시 소스는 셋(`.gitignore` / `.git/info/exclude` / 전역 `core.excludesFile`)인데, `.gitignore`만 봐선 뒤 둘을 놓친다.
- **해결**: 공유할 파일만 **커밋되는 `.gitignore`**로 규칙을 옮긴다. 폴더 통무시(`.obsidian/`)가 아니라 **내부 선별 무시** — `.obsidian/*` + `!.obsidian/app.json`. (디렉토리 통무시 `.obsidian/`는 git이 그 안으로 순회를 안 들어가 하위 `!` 예외를 무력화하므로 `/*`+`!` 쌍이 필수다.) 이미 exclude로 가려져 있던 파일은 `git add -f`로 강제 추적을 시작한다 — 한 번 tracked면 이후 무시 규칙과 무관하게 유지된다. 다른 기기는 `git pull`하면 딸려온다.
- **재발방지**: "왜 이 경로가 무시되지?"는 `.gitignore` grep이 아니라 **`git check-ignore -v <경로>`**로 출처(파일:줄)를 먼저 짚는다 — 로컬 전용인 `.git/info/exclude`·전역 `core.excludesFile`은 clone에 안 실리니, 기기 간 공유가 필요한 설정을 여기에 두면 다른 기기에서 조용히 사라진다. 공유 대상은 반드시 커밋되는 `.gitignore` + 추적 파일로 다룬다. (실측 2026-07-18: PR #50에서 `.obsidian/app.json`만 이 방식으로 공유.)
