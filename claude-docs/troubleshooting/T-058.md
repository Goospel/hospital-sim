# T-058 · 대회 요강을 저장소 요약으로 대리하다 — 제출물 정의가 원문과 어긋난 채 6일

- **증상**: 사용자가 요강의 **제출물 5종 표**를 캡처해 보여주며 "이 5개인 걸 알고 진행한 거야?"라고 물었다. 대조하니 어긋난 게 셋이었다 — ⓐ 5번의 실제 이름은 「**팀원 롤 기술서**」인데 저장소 문서 두 곳(`submission-plan.md`·`final/README.md`)엔 "팀 소개 문서"로 적혀 있었고, ⓑ 표에 없는 「**포트폴리오(선택)**」가 "스킵" 항목으로 적혀 있었으며, ⓒ 1번 비고의 `GitHub Pages (링크)`를 **아무도 본 적이 없었다**(현재 배포는 Vercel). 개수·범위는 우연히 맞았다(솔로라 5번 생략 → 1~4번). 그래서 **틀린 줄 모른 채 ③④ PDF를 만들고 제출 폴더까지 신설**했다.
- **원인**: 요강 원문은 저장소 밖(대회 페이지)에 있고, 그 요약이 부트스트랩 때 `submission-plan.md`에 손으로 옮겨졌다. 이후 모든 판단이 **그 요약을 원문의 대리물로** 삼았다 — 나는 저장소 문서를 읽고 "필수 4종"을 사실로 다뤘다. 요약은 한 번 적히면 **원문과 갈라져도 아무 신호를 내지 않는다**(외부 원본이라 diff가 불가능하고, 저장소 안에서는 완전히 일관돼 보인다). 게다가 `plan.md`의 선결 항목 「제출 요강/폼 전문 정독」은 **⬜인 채로 6일간 남아 있었다** — 즉 "확인 안 했음"이 문서에 정확히 적혀 있었는데도, 그 미확인 상태가 **다운스트림 작업을 멈추지 않았다.** 이 저장소가 이미 두 번 적은 병의 세 번째 판본이다(「AI 활용 추적」·「changeLog 규약」): *규약은 불이행이 아니라 **읽는 사람이 없어서** 죽는다.* 여기서 죽은 건 규약이 아니라 **미확인 표시**였다.
- **해결**: 요강 원문 표를 **전사한 단일 출처**를 만들었다([docs/submission/requirements.md](../../docs/submission/requirements.md)) — 5종 표 + 진행 체크리스트 + **아직 확인 못 한 것** 절. 목록을 다시 적던 문서 넷(`plan.md`·`submission-plan.md`·`final/README.md`·`README.md`)은 **링크만** 남기고 목록을 지웠다(이중 기재 제거). [CLAUDE.md](../../CLAUDE.md) 최상단에 이 문서를 필독 단일 출처로 인덱싱하고, **언제 읽는지**(제출물에 영향 주는 작업 전·제출 PR 때·요강 재확인 때)를 같이 못박았다. 자동 리마인더로 `npm run pdf`가 그 체크리스트를 파싱해 **미완료 제출물과 미확인 항목을 매번 콘솔에 찍게** 했다([scripts/build-submission-pdfs.mjs](../../scripts/build-submission-pdfs.mjs)) — 제출 문서를 건드리면 반드시 지나는 경로다.
- **재발방지**: **저장소 밖에 원본이 있는 요구사항(대회 요강·API 계약·법정 요건)은 요약하지 말고 전사하고, 전사본임을 문서에 명시한다** — 요약은 원문과 갈라져도 신호를 못 낸다. 그리고 **미확인 항목은 문서 안에 `❓`/미체크로 남겨 "모른다"가 산출물이 되게 한다** — plan의 ⬜ 하나로는 부족했다. 결정적 교훈: **"확인 안 함"이 어딘가에 적혀 있는 것과, 그것이 작업을 멈추는 것은 다르다.** 미확인 표시를 **작업 경로 위에**(여기선 빌드 스크립트 출력) 올려야 실제로 읽힌다. 넓게는 [T-057](T-057.md)과 같은 계열 — 관측되는 자리(저장소 문서는 일관됨)와 원인의 자리(원문과의 괴리)가 어긋나면 내부 정합성 검사로는 영원히 안 잡힌다.
