# T-059 · Git Bash가 `/로-시작하는-env-값`을 Windows 경로로 바꿔치기한다

- **증상**: GitHub Pages용 정적 export를 로컬에서 재현하려고 Git Bash에서 `PAGES_BASE_PATH=/hospital-sim npx next build`를 돌리자 빌드가 죽었다.
  ```
  Error: Specified basePath has to start with a /, found "C:/Program Files/Git/hospital-sim"
  ```
  `/hospital-sim`을 분명히 넘겼는데 `C:/Program Files/Git/hospital-sim`이 도착했다. **에러 메시지가 "슬래시로 시작해야 한다"고 말해서 설정 코드를 의심하게 만든다** — 실제로 설정은 멀쩡했고 값이 셸에서 이미 바뀌어 있었다.
- **원인**: MSYS/Git Bash의 **자동 경로 변환**. 네이티브(비-MSYS) 실행 파일에 인자·환경변수를 넘길 때, 유닉스 절대경로처럼 보이는 문자열(`/foo`)을 Windows 경로로 번역해 준다 — POSIX 도구를 Windows에서 쓰라고 만든 편의 기능이다. `/hospital-sim`은 실제 경로가 아니라 **URL의 하위 경로**인데 셸이 그걸 구분하지 못하고 MSYS 루트(`C:/Program Files/Git`)를 앞에 붙였다. 「Windows 셸 한글·인코딩 원칙」(글로벌 CLAUDE.md)의 1·2번과 같은 계열 — *셸이 값을 조용히 고쳐놓고 에러는 downstream에서 난다*.
- **해결**: PowerShell로 돌린다(변환 자체가 없다).
  ```powershell
  $env:PAGES_BASE_PATH = "/hospital-sim"; npx next build
  ```
  Git Bash를 고집한다면 `MSYS_NO_PATHCONV=1`을 앞에 붙이거나 값을 `//hospital-sim`으로 이스케이프한다. **CI는 영향 없다** — GitHub Actions 러너는 Linux라 변환이 존재하지 않고, 실제 배포 경로는 워크플로가 `PAGES_BASE_PATH: /${{ github.event.repository.name }}`로 넘긴다([.github/workflows/deploy-pages.yml](../../.github/workflows/deploy-pages.yml)). 즉 **이 함정은 "로컬에서 CI를 재현하려는 사람"만 밟는다.**
- **재발방지**: **슬래시로 시작하는 값(URL 경로·정규식·컨테이너 내부 경로)을 Git Bash에서 네이티브 exe에 넘기지 않는다** — 이 머신에서는 PowerShell이 기본이다. 그리고 값이 이상하게 도착했을 때 **받는 쪽 코드를 먼저 의심하지 말 것**: 에러 메시지는 도착한 값만 알지 누가 바꿨는지 모른다. 한 단계 앞에서 `echo`로 실제 전달값을 찍는 게 빠르다.
