# T-062 · 지웠는데 되살아나고 클릭할수록 늘어난다 — 옵시디언이 만드는 0바이트 유령 노트

- **증상**: 저장소 루트에 정체 모를 **0바이트 md**가 쌓여 있었다 — `EPILOGUE.md`, `.env.example.md`, `worktree-preview-runs-in-main-repo.md`, 그리고 추적 중인 정본과 **경로만 다른** `troubleshooting/T-056.md`·`superpowers/specs/2026-07-20-hospital-growth-system-design.md`(정본은 각각 `claude-docs/`·`docs/` 아래에 멀쩡히 있다). 지웠더니 `git status`까지 깨끗해졌는데 **1분 뒤 같은 파일이 새 타임스탬프로 되살아났다.** 게다가 `docs/research`를 가리키는 링크는 **클릭할 때마다 파일이 하나씩 늘었다** — `research.md` → `research 1.md` → `research 2.md` → `research 3.md`.
- **원인**: **옵시디언은 해석되지 않는 링크를 클릭하면 그 이름으로 빈 노트를 만든다.** 저장소가 곧 볼트라 그 산출물이 git 작업 트리에 그대로 떨어진다. 두 갈래였다.
  1. **폴더를 가리키는 마크다운 링크** — [README.md](../../README.md) 문서 내비게이터의 `[docs/research/](docs/research/)`·`[docs/submission/final/](docs/submission/final/)`. 노트가 아니라 **디렉터리**라 옵시디언은 이걸 **영원히** 해석하지 못한다. 그래서 한 번 만들고 끝이 아니라 클릭마다 새로 만들고, 이름이 겹치니 ` 1`·` 2`를 붙인다 — **클릭 횟수만큼 증식하는 구조**다. GitHub에서는 같은 링크가 정상 동작해서 **아무 문제 없어 보인다**는 게 함정의 핵심이다.
  2. **세션 복원** — `.obsidian/workspace.json`의 `lastOpenFiles`에 남은 항목을 볼트가 다시 열면서 없어진 파일을 0바이트로 되만든다. 그래서 파일만 지우면 **삭제가 안 먹는 것처럼** 보인다(실제로는 지워졌고, 그 뒤 재생성된 것이다). `.env.example.md`처럼 md가 아닌 대상(`[.env.example](.env.example)`)에 옵시디언이 `.md`를 붙여 만든 것, `[[dangling-link]]`(메모 규약이 **일부러** 허용하는 미작성 링크)를 클릭해 생긴 것도 같은 갈래다.
- **해결**: 원인별로 따로 끊는다.
  - **폴더 링크를 없앤다** — 내비게이터 표에서 링크를 **경로 표기(백틱)**로 내렸다. GitHub 클릭 편의를 잃지만, 잃지 않으면 클릭할 때마다 쓰레기가 쌓인다. 파일을 가리키는 링크는 그대로 둔다(정상 해석된다).
  - **볼트 안에서 지운다** — 이미 생긴 유령 노트는 옵시디언 파일 탐색기에서 **우클릭 → 삭제**로 없앤다. 파일시스템에서만 지우면 `lastOpenFiles`에 남아 세션 복원 때 되살아난다.
  - **정본 확인 후 삭제** — 경로만 다른 사본은 **0바이트인지, 정본이 추적 중인지**를 먼저 확인하고 지운다(여기선 T-056 3,433 bytes·spec 14,602 bytes가 각각 정상 추적 중이었다).
- **재발방지**: **마크다운 링크가 폴더를 가리키지 않게 한다** — 저장소가 옵시디언 볼트를 겸하면 링크는 문서 장식이 아니라 **파일을 만드는 부작용**을 갖는다. 넓게는 *"같은 텍스트가 두 렌더러에서 다르게 해석되고, 한쪽에서만 부작용이 있다"*는 유형이라 **GitHub에서 멀쩡한 것이 검증이 되지 못한다**. 그리고 이건 [CLAUDE.md](../../CLAUDE.md) 「옵시디언 문서 태그」가 세운 원칙 — *"검사기 없는 규약은 죽는 게 아니라 썩는다"* — 이 그대로 적용되는 자리다: "폴더 링크 금지"를 소프트 규칙으로 두면 다음 문서에서 조용히 재발하고, 증상이 **저장소가 아니라 남의 앱(옵시디언)에서** 나타나 더 늦게 발견된다. **그래서 검사기를 붙였다** — [check-doc-tags.ps1](../../scripts/check-doc-tags.ps1)에 규칙 2로 들어가 md를 건드리는 커밋마다 돈다(TDD·[test-check-doc-tags.ps1](../../scripts/test-check-doc-tags.ps1) 케이스 5~7). 붙이자마자 **내 수동 스캔이 놓친 3건**([plan.md](../plan.md) 1건 · [requirements.md](../../docs/submission/requirements.md) 2건)을 잡았다 — 사람이 훑는 것과 매 커밋 도는 검사기의 차이가 그대로 나왔다. 검사기에서 **코드 스팬·코드블록은 제외**해야 한다: 안 그러면 이 규칙을 설명하는 문서(이 T-062)가 자기 자신에게 거부당한다. 계열: [T-061](T-061.md)(중간 계층이 값을 바꾸면 하류 전제가 깨진다) · [T-056](T-056.md).
