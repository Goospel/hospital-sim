# T-073 · 페인 미표시 환경에서 합성 클릭이 안 먹고 타이머 렌더가 안 flush됐다

- **증상**: SDD 브라우저 검증 중 ⓐ `computer` 도구의 클릭이 페이지에 전혀 전달되지 않았고(같은 세션 초반엔 되던 것), ⓑ 응급 카드의 15초 카운트다운이 화면에서 14초→5초까지 줄다가 **완전히 멈췄다**. 처음엔 Task 1의 prop 배선 회귀(T-043류)나 카운트다운 구현 결함으로 의심됐다.

- **원인**: 둘 다 코드 결함이 아니라 **브라우저 페인이 표시되지 않는(hidden) 환경**의 아티팩트다. `document.hidden === true`이고 페이지가 컴포지팅하지 않는다. ⓐ 합성 클릭은 컴포지팅이 없으면 전달되지 않는다. ⓑ **타이머는 정상으로 돈다**(직접 만든 `setInterval`이 8초에 6회 틱) — 멈춘 것은 **React가 타이머發 상태 갱신을 flush하지 않는 것**이다. 반면 이산 이벤트(클릭)發 갱신은 동기 우선순위라 정상 반영된다.

- **해결**: 검증을 `javascript_tool`의 `element.click()`으로 구동한다(React 핸들러 경로는 동일해 배선 검증에 유효). 타이머 기반 UI의 실동작은 이 환경에서 관측 불가이므로, **React 파이버를 직접 읽어**(`el[Object.keys(el).find(k=>k.startsWith('__reactFiber$'))]`를 타고 올라가 `memoizedProps`) 상태가 실제로 진행했는지 판별한다 — 이번엔 그 방법으로 "TIMEOUT이 조용히 발화한 게 아니라 렌더가 멈춘 것"임을 확정했다(`receiving.index`가 그대로였다).

- **재발방지**: 브라우저 검증이 이상하면 **먼저 `document.hidden`과 자체 타이머 틱을 재어 환경을 배제**한 뒤 코드를 의심한다. 타이머 기반 UI의 만료·애니메이션은 이 환경에서 자동 검증 대상으로 잡지 말고, 코어 단위 테스트(결정론) + 이산 이벤트 실측으로 나눠 덮는다.
