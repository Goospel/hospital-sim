# T-054 · 워크트리 preview는 메인 레포 기준으로 돌아 변경이 안 보인다

- **증상**: `.claude/worktrees/...` 워크트리에서 코드를 고쳤는데, `preview_start`로 띄운 프리뷰엔 반영이 안 되고 옛 화면이 나온다. 워크트리에서 한 변경을 브라우저로 시각검증하려는데 계속 이전 상태만 보인다.
- **원인**: `preview_start`(및 그 아래 `next dev`)가 **여러 lockfile을 감지**해 워크스페이스 루트를 **메인 레포**로 추론한다(Next 경고: "inferred your workspace root … selected the directory of `<메인레포>/package-lock.json`"). 그러면 메인 레포 기준으로 서빙/해석해 워크트리(브랜치)의 변경이 안 보인다.
- **해결**: 워크트리 변경의 시각검증은 `preview_start`에 의존하지 말고 **워크트리 cwd에서 직접 `npx next dev -p <메인과 다른 포트>`**(예: 3100)를 백그라운드로 띄우고, 그 `localhost:<포트>`를 브라우저로 연다. 서빙 코드가 실제로 워크트리 것인지 **DOM으로 먼저 확증**한다(예: 방금 바꾼 className `max-w-2xl`를 `getComputedStyle`/`classList`로 실측). `turbopack.root` 경고가 "메인 레포 package-lock 선택"으로 뜨는 게 이 함정의 신호다.
- **재발방지**: 워크트리 작업의 브라우저 검증은 "어느 서버가 어느 소스를 서빙 중인가"를 **먼저** 확인하고 시작한다. 같은 pane 계열 함정 [T-053](T-053.md). (원래 auto-memory `worktree-preview-runs-in-main-repo`에 있던 것을 트랩 성격에 맞게 이관 — 실측 2026-07-20, PR #70·#71 검증 중 재확인.)
