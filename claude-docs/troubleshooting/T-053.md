# T-053 · 브라우저 pane이 hidden이면 하이드레이션이 멈춰 인터랙션 검증 불가

- **증상**: 브라우저 pane에서 버튼 클릭이 화면을 안 바꾼다. `computer{left_click}`도, `javascript_tool`로 준 프로그램적 `btn.click()`도 상태를 안 바꾼다. `useEffect`가 안 돈다(예: 마운트 fade-in의 `opacity`가 0에 멈춤). `computer{screenshot}`은 30초 타임아웃. `read_console_messages`·`read_network_requests`는 빈 값. 그런데 DOM엔 React fiber·props·`onClick` 함수가 붙어 있어(`__reactFiber$`/`__reactProps$` 존재) "내 코드 버그"로 오진하기 쉽다.
- **원인**: pane 탭이 `document.visibilityState === "hidden"`(백그라운드)라 크롬이 탭을 throttle → React 동시성 스케줄러(MessageChannel/timer 기반)가 멈춘다. 하이드레이션은 **시작(fiber attach)까지만** 되고 **commit·effect 플러시가 안 돼** 앱이 인터랙티브가 안 된다. 즉 코드가 아니라 **탭이 안 보이는 환경**이 원인. (스크린샷 타임아웃·콘솔/네트워크 빈 값도 같은 뿌리 — 렌더러가 페인트를 안 함.)
- **해결**: 클릭이 안 먹으면 **가장 먼저 `javascript_tool`로 `document.visibilityState`/`document.hidden` 확인**. `hidden`이면 환경 문제다. `tabs_select`로도 안 풀리고, pane을 실제로 화면에 띄우는 건 하네스/사용자 몫이라 이 세션에선 끝내 못 풀었다. **DOM 읽기(`read_page`·`javascript_tool`)는 hidden에서도 동작**하니(SSR HTML은 DOM에 있음), 특정 화면 컴포넌트를 검증하려면 **초기 상태를 그 페이즈로 임시 시드**(예: `useState(() => enterWorldEvent(startSession()))`)하고 기본 접힘 UI는 `useState(true)`로 열어 **SSR 강제 렌더** → DOM을 읽어 마크업·텍스트·순서·`aria`를 실측한다. 클릭 토글 자체(하이드레이션 필요)는 여전히 검증 불가 → 그건 Vercel 미리보기로 넘긴다.
- **재발방지**: browser-pane에서 인터랙션이 안 먹으면 **코드 bisect 전에 `visibilityState`부터** 본다(이 세션은 무죄 입증에 bisect까지 갔다). 부수 함정: 워크트리에서 `next dev`를 `&`로 띄운 뒤 TaskStop해도 자식 node가 포트를 붙든 좀비로 남을 수 있으니, 포트 충돌 시 `netstat -ano | grep :PORT` → `taskkill //PID <pid> //F`로 정리. 같은 pane 계열 함정 [T-054](T-054.md). (실측 2026-07-20, PR #71 용어 풀이 검증 중.)
