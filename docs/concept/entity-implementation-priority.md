# 미구현 도메인 엔티티 구현 우선순위

> **목적**: [entity-code-coverage.md](entity-code-coverage.md)가 판정한 **⬜ABSENT 51개**(+🟡PARTIAL 심화 후보) 중, 구현 시 게임 핵심 메시지에 가장 크게 기여할 후보의 우선순위를 매긴다. 무엇이 *안* 구현됐나(coverage)에서 *다음에 무엇을* 구현하나(priority)로 넘어가는 다리.
> **방법**: [game-concept.md](game-concept.md)(기준축)·[domain-entities.md](domain-entities.md)(카탈로그·문제→엔티티)·[entity-code-coverage.md](entity-code-coverage.md)(반영도)·[medical-system-grounding.md](../research/medical-system-grounding.md)(사실 근거) + `src/game/*.ts` 전량 통독. **show-don't-tell·톤 가드레일을 상단 필터**로 적용. 내부 독립 채점 패널 3인 + 적대 검증을 입력으로 쓰되, 모든 정수 점수는 코드 증거로 직접 확정(패널 평균 미채택 — 검증관이 반증한 F 과대평가를 하향 반영). 2026-07-19.
> **판정 근거**: 코드·문서 증거 + 내부 독립 채점 패널 3인 + 적대 검증. 단일 세션 산출이라 M·C 판단엔 ±1 여지가 있다(아래 §4 확신도). 재검토·별도 판정과의 대조는 §재현을 따른다.

## 한 줄 결론

**이미 있는 훅에 데이터/인과만 배선하면 새 병목이 창발하는 세 후보** — ①거리·이송 축(축③ 부활) ②재정중립(개혁 무력의 기전화) ③방어진료(소송×검사 인과) — 이 저비용·고메시지·팩트안전의 삼박자로 최상위다. 저수가 소청/산부 시나리오는 게임 문서가 지목한 본선 정본. 반대로 실명 사건·의정갈등은 아무리 '중요'해도 톤 가드레일이 **영구 배제**한다.

## 채점 방법 (고정 정의)

**우선순위 점수 = M×2 + F + C + S (25점 만점, 메시지에 가중).**

| 축 | 정의 | 5점 | 1점 |
|---|---|---|---|
| **M** 메시지 기여도 | 게임 핵심 메시지(한국 응급의료의 구조적 문제를 플레이어가 '겪게')를 얼마나 날카롭게 하는가 | 핵심 병목을 **새로 드러냄** | 주변적 |
| **F** 메커닉 적합성 | 기존 축(backupCare·수가 델타·골든타임·장부·외생이벤트)에 얼마나 자연스럽게 붙나 | 기존 게이트에 **바로** | 새 서브시스템 필요 |
| **C** 구현 비용 낮음 | 낮을수록 고점 | 필드/상수 몇 개 | 새 페이즈·UI·상태기계 대공사 |
| **S** 팩트·톤 안전 | 리서치 근거 + 톤 가드레일(실명·비난·정파·미검증수치 금지) 지키기 쉬운가 | 안전 | 오형상화·2차가해 위험 |

**상단 필터(강등 규칙)**: show-don't-tell·톤 가드레일 위반을 요구하는 엔티티(실명·정파 지목·해석 카피)는 '중요'해도 S 강등. `medical-system-grounding.md`의 검증실패 수치 목록(700명 입건·90% 병상가동률·소청과 28%↓·여론조사 등)을 화면에 노출해야 하는 엔티티도 S 강등.

**부착점 판단의 기준(코드 사실)**:
- `round.ts:42 attemptTransfer(state, hospitalId, timeCostSeconds)` — 병원별 이송시간 **파라미터가 이미 존재**(현재 균일값)
- `world.ts WorldEvent.effects: DeptEffect[]` — **이미 다효과 배열**(단 `field`는 profit/hireCost로 제한 — 헌법)
- `receiving.ts` `lawsuitExposure`(수용 시 누적)·`workup`(검사 흑자·boarding)·거절 로직이 **모두 공존**
- `types.ts Specialty`에 CARDIOLOGY·THORACIC·OBSTETRICS·NEUROSURGERY·GENERAL_SURGERY 이미 존재(2막은 STEMI=CARDIOLOGY만 요구, 나머지 배후 게이트는 **휴면**)
- `news.ts FORBIDDEN_REAL_EVENT_TOKENS` — 실명 사건 토큰을 코드로 영구 차단
- `session.ts EmergencyState` — IN_HOUSE|TRANSFER **2모드뿐**

## 2. Top 8 상세

### 1위 · 응급의료취약지 = 거리·이송 축 (문제③) — 22점
- **(a) 무엇을**: 병원마다 `distanceMinutes`를 부여해 전원 시도 시간에 반영 — 일부 PCI 병원은 골든타임 반경 밖(도달 불가). "가까운 곳엔 순환기가 없고, 순환기 있는 곳은 60분 밖".
- **(b) 어디에**: `round.ts:42 attemptTransfer(state, hospitalId, timeCostSeconds)` — **파라미터가 이미 존재**(현재 균일값). `scenarios.ts` 병원에 거리 필드 추가 + `goldenTime` 카운트다운 재사용. 새 서브시스템 불요.
- **(c) 왜 큰가**: 3대 문제 중 **유일하게 메커닉이 0인 축③**을 살린다. grounding §5가 지방공백의 진짜 킬러를 '수용 거부'가 아니라 **'반경 안에 받을 병원이 애초에 없다'**로 지목(취약지 98곳·평균 141분·사망률 3배). 배후병목(NO_BACKUP_CARE)과 **다른 종류의 벽**을 새로 드러낸다.
- **(d) 절단선**: 최소형(병원별 거리 → 도달불가 잠금)은 **MVP 편입 가능**. 간이 지도·119 병원전 단계는 본선.

### 2위 · 재정중립 (문제⑤) — 21점
- **(a) 무엇을**: "정책수가 +6을 주면 재정중립 제약으로 다른 과가 −6" — 개혁 이벤트에 상쇄항을 짝지어 순효과를 0에 수렴시킨다.

- **(c) 왜 큰가**: "왜 저수가가 안 고쳐지나"를 **부족이 아니라 구조(상쇄)**로 만든다. 기존 `CARDIO_FEE_HIKE` 주석("미용 +70이 압도")이 이미 암시하던 걸 **기전으로** 굳혀, §7 엔딩 철학('쟁취 아닌 필연·정답 퍼즐 방지')을 정책 층에서 강화.
- **(d) 절단선**: 이벤트 카탈로그에 한 항목 추가라 **MVP 가능**. 단 정책 층이라 뺑뺑이만큼 체감적이진 않다.

### 3위 · 방어진료 (문제⑥/②) — 20점
- **(a) 무엇을**: 고위험 케이스는 (거절해 소송 노출 회피) 또는 (수용하되 방어적 검사 남발)의 인과를 명시 — 지금은 두 축이 따로 논다.

### 4위 · 소아청소년과/산부인과 저수가 시나리오 (문제②) — 19점

## 4. 확신도 & 총평

**확신도: 중상(中上)** — 상위 8개의 **형태·부착점·톤 안전은 상**(코드에서 훅 존재를 직접 확인: `timeCostSeconds`·`effects: DeptEffect[]`·`lawsuitExposure`+`workup`; 검증관이 소청과·특례법·형사경로·전원협약의 F 과대를 반증해 하향 반영). **불확실은 C축**(구현 비용은 '어디까지 최소화하느냐'의 판단이라 ±1)과 **14점 평지의 세부 순위**(유의미하지 않음).

**총평**: 이미 있는 훅에 데이터/인과만 배선하면 새 병목이 창발하는 세 후보 — ①거리·이송 축(축③ 부활) ②재정중립(개혁 무력의 기전화) ③방어진료(소송×검사 인과) — 가 저비용·고메시지·팩트안전의 삼박자로 최상위이고, 저수가 소청/산부 시나리오는 게임 문서가 지목한 본선 정본이다. 반대로 실명 사건·의정갈등은 아무리 '중요'해도 톤 가드레일이 영구 배제한다.
