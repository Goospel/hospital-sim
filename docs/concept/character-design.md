# 캐릭터 디자인 — 의사 · 환자

> **목적**: 세션이 끊기고 새로 시작해도 같은 캐릭터가 나오게 한다.
> 이 문서를 읽는 사람은 대개 **다음 세션의 나(Claude)**다 — 캐릭터를 만지기 전에 여기부터 본다.
> 관련: [game-concept.md](game-concept.md) · [domain-entities.md](domain-entities.md)

## 0. 경계 — 값은 코드에, 이유는 여기에

**색 코드·픽셀 격자·크기 값을 이 문서에 복붙하지 않는다.** 그 순간 한쪽이 조용히 낡는다(CLAUDE.md가 세 번 경고한 이중 기재 병).

| 무엇 | 단일 출처 |
|---|---|
| 스프라이트 격자·팔레트·과별 색 | `src/components/PixelSprite.tsx` |
| 표시 크기·배치·조명 워시 | `src/components/HospitalMap.tsx` |
| 배회·보행 애니메이션 | `src/app/globals.css` (`hm-wander` · `hm-walk`) |
| 아바타의 존재·위치 규칙(파생) | `src/game/hospitalMap.ts` (`deriveMapScene`) |
| 의사 이름 생성 | `src/game/doctor.ts` (`doctorName`) |
| 발신자 대사 | `src/game/dialogue.ts` (`CALLER_PLEA`) |

**이 문서가 유일한 출처인 것**: 아래 §1~§6 — 원칙 · 금지 · 추가 절차. 코드는 "무엇"만 말하고 "왜 그렇게만 하는가"는 안 적혀 있다.

## 1. 공통 형식 (의사·환자 공유)

- **8×8 픽셀 그리드를 inline SVG `<rect>`로 그린다.** 이미지 파일 0개 → 번들 영향 0, 정적 export(GitHub Pages) 안전.
- **외부 스프라이트·타일셋을 추출해 쓰지 않는다.** ZEP·상용 에셋 모두. 대회 제출물이라 실제 저작권 리스크다. 참조하는 것은 **형식**(2D 탑다운 + 픽셀 그리드)까지이고, 이는 Gather.town·스타듀밸리·테마 호스피탈이 공유하는 장르 관습이다.
- **실루엣은 하나(`FIGURE`)를 공유하고 팔레트만 다르다.** 8×8에서 형태를 나누면 양쪽 다 뭉개진다. 그리고 의사와 환자가 같은 몸을 쓰는 건 이 게임 주제와 맞는다 — 다른 종족이 아니라 **같은 사람이 어느 자리에 있느냐**의 문제다.
- **역할 구분은 색으로만 한다.** 아이콘·배지·말풍선·이름표를 얹지 않는다.

## 2. 의사

- **흰 가운 + 과 색 어깨.** 과 색 팔레트 규칙: **필수과는 채도 있는 색, 수익과는 무채색 회색.** 미용·검진 의사가 회색인 건 우연이 아니라 대조 장치다 — "수익과는 회색입니다"라고 쓰지 않고 색만 놓는다(show-don't-tell).
- **상태는 배지가 아니라 밝기.** 진료 중이면 가운이 밝아진다. 상태점·아이콘을 새로 붙이지 않는다.
- **위치가 곧 상태다.** 진료실 안 = 진료 중, 복도 = 자유. `deriveMapScene`이 `busyUntil` 하나에서 파생하므로 표시용 상태를 따로 만들지 않는다.
- **이름은 결정론 한글 이름**(`doctorName`, RNG 0). 같은 선택은 항상 같은 명단이다.
- **성격·사연·초상화는 없다.** 의사는 개인이 아니라 **자리 하나**다. "번아웃한 순환기 김 선생의 사연"을 넣는 순간 시스템 이야기가 인물 드라마가 된다.
- 유일한 개체 단위 상태 표현은 담당 수와 피로 막대(`DoctorRoster`)이고, 그것도 숫자·막대뿐 해석 문구가 없다.

## 3. 환자

- **익명 회색. 이름도 나이도 성별도 사연도 없다.** 도메인 타입 `Patient`가 갖는 건 `requiredSpecialty`와 `severity`뿐이다.
- **이것은 의도적 영구 보류다.** 개인 서사가 붙으면 "**시스템이** 문제"가 "**이 환자가** 안됐다"로 미끄러진다. 플레이어를 울리는 게 목적이 아니라 구조를 보게 하는 게 목적이다.
- **환자는 말하지 않는다.** 목소리를 내는 건 환자가 아니라 **전화 건 쪽(다른 병원 담당자)**이다(`CALLER_PLEA`). 톤은 명랑한 시스템 대사와 인간의 대가 사이의 낙차이고, **과녁은 시스템의 태연함이지 환자도 거절하는 의사도 아니다.**
- **화면에 나오는 회색 인물은 두 종류이고 섞으면 안 된다**:
  - **침대 위 환자** — 게임 상태(`busyUntil`) 파생. 진료 중인 의사 1명 = 환자 1명.
  - **배경 보행자** — 순수 장식. 게임 상태가 아니고(`MapScene` 밖), 불투명도를 낮추고 `aria-hidden`이다. 이걸 `avatars`에 섞으면 「보이는 것 = 게임이 모델링하는 것」이 깨진다.

## 4. 움직임

- **세 층이 서로 다른 CSS 속성을 쓴다** — 겹치면 아바타가 튄다:
  - 배치 이동(방↔복도↔침대): 바깥 div의 `left`/`top` + transition
  - 유휴 배회: 안쪽 div의 `transform`
  - 배경 보행: `left` 키프레임(별도 레이어)
- **아바타마다 박자가 다르다**(`wanderTiming`). 전원이 같은 주기로 흔들리면 사람이 아니라 기계로 보인다. 박자도 id 해시 파생이라 결정론이다(RNG 0 원칙은 표시 레이어에도 적용된다).
- **연출 층은 게임 시계와 분리돼 있다.** 플레이어가 콜을 노려보느라 시계가 멈춰도 병원은 계속 돈다.
- **`prefers-reduced-motion`은 필수 경로다.** 배회는 정지, 배경 보행자는 통째로 숨김(애니메이션만 끄면 다섯 명이 왼쪽 끝에 겹쳐 선다).

## 5. 캐릭터를 손대기 전 체크리스트

1. **새 실루엣이 정말 필요한가** — `FIGURE` + 팔레트 교체로 안 되는가? (간호사·보호자·구급대원 전부 팔레트로 된다)
2. **값을 `PixelSprite.tsx` 밖에 쓰려는가** — 컴포넌트에 색을 하드코딩하지 않는다.
3. **개인 서사(이름·나이·표정·초상)를 붙이려는가** — §2·§3 위반. 하지 않는다.
4. **해석 문구를 붙이려는가** — "과로 상태입니다" 같은 카피 금지. 사실만 놓는다(메모 `game-show-dont-tell`).
5. **외부 에셋인가** — 금지(§1).
6. **새 상태를 배지로 표현하려는가** — 먼저 밝기·위치로 되는지 본다.
7. **게임 상태를 새로 만드는가** — 표시층은 파생만 한다. 판정·경제에 0 침습.
8. **reduced-motion에서 깨지지 않는가.**

## 6. 하지 않기로 한 것 (의도적 보류 — 새 세션이 되살리지 말 것)

| 안 하는 것 | 왜 |
|---|---|
| 환자 이름·나이·사연·초상 | 시스템 문제가 개인 신파로 미끄러진다(§3) |
| 의사 표정·감정 애니메이션 | 8×8에서 안 읽히고, 읽히면 인물 드라마가 된다 |
| 4방향 스프라이트·걷기 프레임 | 위치 변화만으로 충분하다. 프레임이 늘면 팔레트 규칙이 8배로 늘어난다 |
| 외부 에셋·AI 생성 이미지 | 저작권 리스크(제출물) + 픽셀 격자의 일관성이 깨진다 |
| 캐릭터 초상화·프로필 UI | 개체에 시선이 모이면 대조(필수과 vs 수익과)가 안 보인다 |
| 환자 대사 | 환자는 말하지 않는다 — 목소리는 발신자 몫(§3) |
