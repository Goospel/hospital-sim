# AI 활용 방향 재검토 — "실시간 대화"에서 "세계를 재구성하는 뒷단 AI"로

> **상태**: 방향 탐색 초안 (2026-07-18). 사용자 아이디어를 Claude가 이해한 대로 옮긴 것 — **확정 설계 아님.**
>
> **1차 결정 (2026-07-18)**
> - §5-1 = **세계가 자율적으로 변한다** — 수가·법 변경은 외생 이벤트로 떨어지고, 플레이어는 그 속에서 병원을 운영·적응한다(지금의 '겪는 자' 시점 유지).
> - §5-3 = **새 축으로 대체** — 실시간 대화형 전원 협상을 폐기하고, **뒷단 세계 재구성이 AI의 핵심 축**이 된다.
>
> **2차 결정 (2026-07-18)**
> - §5-A = **기존 전원 시스템 유지 + 자유 텍스트 입력 박스만 삭제.** 전화할 때 말솜씨는 뺑뺑이 체감과 무관하므로(서비스업이 아님) 뺑뺑이 미니게임 골격(골든타임·병원 리스트·거절 사유)은 그대로 두고, 플레이어 설득 입력창만 제거한다 → 대사 LLM(`dialogue.ts`)은 실질 폐기.
> - §5-B = **라이브 AI 노출은 포기.** 뒷단 생성은 영상에서 안 보이는 게 구조적 한계 → 라이브 반증 장치를 새로 만들지 않고, **디렉팅·아키텍처를 문서(ai-usage-doc)로 강조**하는 전략으로 간다. 영상에 AI가 안 드러나는 단점은 감안.
>
> **3차 결정 (2026-07-18)**
> - §5-2 = **데이터 + 서사 둘 다.** AI가 병원 파라미터(JSON)를 바꿔 판정에 직결시키고, 그 변화를 신문·지표 서사로 플레이어에게 보여준다(기존 '이번 주 신문' UI 재사용).
> - §5-4 = **주(회차) 사이마다.** 매 주 시작에 외생 이벤트 1개가 떨어져 세계가 재구성된다. 현재 '한 주=1회차'에 얹히고, N주로 확장.
>
> 확정분은 이후 [game-concept.md](game-concept.md) §7 · [ai-usage-doc.md](../submission/ai-usage-doc.md) §2에 반영한다.

## 0. 한 줄

지금까지 AI는 **플레이어와 실시간으로 대화하는 전원 담당자**였다.
이 전환은 — AI를 대화 상대가 아니라, **현실 세계(법·수가·여론)를 매 주 생성·조절해 게임 세계를 다시 짜는 뒷단 엔진**으로 옮기는 것. 대화형은 **버린다.**

---

## 1. 무엇이 바뀌나 — 두 용도의 대조

| | 지금 계획 (대화형, **폐기**) | 새 방향 (세계 재구성형, **채택**) |
|---|---|---|
| AI가 있는 자리 | 게임 루프 **안**, 플레이어 눈앞 | 게임 루프 **뒷단**, 국면과 국면 사이 |
| AI가 하는 일 | 전원 담당자 **대사를 연기** | 세계의 **상황을 재구성** |
| 입력 | 플레이어의 설득 텍스트 | 세계에 떨어진 **외생 변경**(수가·법 등) |
| 출력 | 담당자 한 마디 | 바뀐 세계 상태(병원들·콜 분포·거절 사유 분포…) |
| 상호작용 형태 | 실시간 채팅 | 세계가 바뀜 → (뒷단 계산) → 달라진 세계를 마주함 |
| 시간 축 | 초 단위(골든타임) | 국면 단위(정책이 바뀐 다음 주) |

---

## 2. 내가 이해한 메커니즘 — 사용자 예시를 구조화

사용자가 든 예시("수가 변경 / 법 변경 → AI가 게임 세상 상황을 구축")를 3단계 루프로 옮기면:

```
① 세계의 현재 상태
   병원 N곳 — 각자 병상 · 응급실 당직 · 배후진료 역량 · (수가로 결정된) 채용 구조
        │
② 변경 이벤트가 외생적으로 떨어진다  ← 세계가 자율적으로 변함 (결정 §5-1)
   예: "순환기 수가 30% 인상"  /  "응급실 수용거부 금지법 시행"
        │
③ AI가 그 변경의 파급을 세계에 반영
   "수가가 오르면 → 순환기를 채용할 여력이 생기는 병원이 늘고
    → 배후진료 가능 병원 수가 바뀌고 → 전원 거절 분포가 달라진다"
        │
   달라진 세계 상태 → 다음 국면의 플레이가 이 위에서 돈다
        │
   플레이어는 그 바뀐 세계 속에서 자기 병원을 운영·적응한다 (겪는 자 시점)
```

**핵심**: AI가 만드는 건 **"정책 → 결과"의 인과 전개**다.
게임의 원래 메시지("구조가 결과를 만든다")와 **같은 축**이되, 이제 그 구조가 **바깥에서 움직이고**(플레이어가 못 막는다), 플레이어는 흔들리는 구조 위에서 병원을 지탱해야 한다.

### 2.1 AI가 조절하는 것 — 법·제도만이 아니라 문화·여론까지

이 게임의 세계관은 **'실제 세상'**이다. 전제: **지금 의료 시스템의 문제는 현실 세계의 법과 문화에서 나왔다.** 그래서 AI가 조절하는 '세계'는 딱딱한 제도 수치에 그치지 않고, 그 수치를 낳는 **사회적 층위**까지 포함한다.

| 층위 | 예시 외생 이벤트 | 병원으로의 파급(파라미터) |
|---|---|---|
| **법·제도** | "응급실 수용거부 금지법 시행" · "순환기 수가 30% 인상" | 거절 제약 · 채용 여력 |
| **문화·여론** | "응급실 사망 기사 → 여론 악화" · "의료소송 판결 화제" | 소송 리스크 · 방어진료 압력 · 특정 과 기피 |

AI는 이 둘을 **기사·법 개정 뉴스라는 서사로 생성**하고(§5-2의 '서사'), 그 서사 뒤에서 **병원 파라미터를 조절**한다(§5-2의 '데이터'). 즉 "기사가 떴다"가 표면, "그래서 순환기가 더 기피된다"가 그 밑의 판정 변수 — 서사와 데이터가 한 이벤트의 앞뒷면이다.

> 이 프레임은 원래 game-concept §6의 톤 원칙("구조를 익명의 덩어리로 두지 않고, 저수가·인력이탈·소송리스크 같은 복수의 구체적 원인으로 병렬 제시")과 정확히 맞물린다 — AI가 매 주 그 '구체적 원인' 하나를 세계에 떨어뜨리는 셈이다.

## 3. 기존 아키텍처와의 관계 — 원칙은 안 깨진다, 계층만 올라간다

[ai-usage-doc.md](../submission/ai-usage-doc.md) §2의 철칙:
**게임 상태 전이(개별 전원 수용/거절)는 절대 LLM이 정하지 않는다. 판정은 코드가 한다.**

새 방향은 이 원칙과 **충돌하지 않는다** — AI가 손대는 자리가 한 계층 위이기 때문이다.

- **AI가 만드는 것**: 세계의 **파라미터** — 병원별 배후진료 역량·병상·수가 반영값 = `adjudicate.ts`가 읽는 바로 그 "숨은 제약 JSON".
- **AI가 여전히 못 만드는 것**: 개별 STEMI 한 명의 수용/거절 **판정**. 그건 AI가 짠 세계 위에서 **여전히 결정론 코드**가 돌린다.

그림으로:

```
[AI: 외생 변경 → 세계 파라미터 재구성]   ← 새 핵심 축 (뒷단 · 국면 사이)
        │  세계 상태(제약 JSON)
        ▼
[코드: adjudicate() 판정]              ← 기존 하드락, 그대로 유지
        │
        ▼
   달라진 세계를 플레이어가 겪는다
   └ 전원 뺑뺑이 미니게임 골격은 유지, 자유 텍스트 입력창만 제거(§5-A 확정)
     → 대사 LLM(dialogue.ts)은 실질 폐기, 전화는 버튼식으로
```

이 배치는 대회의 **디렉팅 서사를 유지·강화**한다:
"AI가 세계를 시뮬레이션하되, 생사 판정만은 코드가 잠근다" — AI에 더 큰 권한(세계 생성)을 주면서도 **끊는 자리는 그대로**. "어디까지 믿고 어디서 끊나"의 선이 더 굵어진다.

## 4. 이 결정이 얻는 것과, 실제로 부담하는 것

**얻는 것**
- 게임의 논지가 **세계 수준**으로 커진다. 지금은 병원 한 곳의 하루를 겪는데, 이 방향은 **제도가 바뀌면 현장이 어떻게 출렁이는지** 전체를 겪게 한다.
- game-concept §7이 꼽은 원래 강점("AI가 무한 변주")을 케이스가 아니라 **세계 수준**에서 실현.
- 판정=코드 원칙 아래에서 AI에 더 큰 역할을 안전하게 위임 → 디렉팅 물증이 두꺼워짐.

**실제로 부담하는 것 (정직하게 — "대체" 결정의 대가)**
- ⚠️ **대회 승부축이 통째로 바뀐다.** 지금까지 설계 전체의 핵심은 *"심사위원이 직접 자유 텍스트로 매달리고, AI가 대본 없이 거절하는 걸 라이브로 겪게 해 'AI 필수성'을 반증시킨다"*였다(ai-usage-doc §2, game-concept §7 line 143). 실시간 대화형을 버리면 **이 라이브 반증 장치가 사라진다.**
- 뒷단 생성형은 game-concept §7 "차선책"이 지적한 약점을 정면으로 안는다 — **"사전 캐시 데모라 '하드코딩 아니냐'는 반박에 취약."** 이 라이브 반증은 **포기하기로 결정**했다(§5-B): 디렉팅·아키텍처를 문서로 강조하고, 영상에 AI가 안 드러나는 단점은 감안한다.
- 세계 재구성을 AI에 맡기면 **의학·경제·사회적 정확성**의 검증 부담이 커진다. 특히 AI가 **여론·기사를 생성**하면(§2.1) [fact-grounding 원칙](../research/medical-system-grounding.md)·톤 가드레일(특정 직군·집단 비난 금지, 선정성 배제)을 그 생성물에도 강제해야 한다 — 자유 생성일수록 위반 위험이 크다.
- MVP 절단선(2026-08-10)이 실시간 협상에 맞춰 잠겨 있다 — 2막(전원 뺑뺑이)·`dialogue.ts`·골든타임 UI의 **폐기/재설계 범위와 남은 일정**을 다시 산정해야 한다.

## 5. 결정 반영 & 남은 질문

**확정됨**
- ✅ **§5-1 촉발 주체 = 세계가 자율적으로 변한다.** 플레이어는 정책 결정자가 아니라, 외생 충격 속에서 병원을 운영·적응하는 '겪는 자'.
- ✅ **§5-3 기존 대화형 = 새 축으로 대체.** 실시간 전원 협상 LLM 대화 폐기.
- ✅ **§5-A 전원 경험 = 기존 뺑뺑이 골격 유지 + 입력 박스만 삭제.** 전화 말솜씨는 뺑뺑이 체감과 무관 → 자유 텍스트 설득 입력창만 제거, 전화는 버튼식. 대사 LLM(`dialogue.ts`) 실질 폐기.
- ✅ **§5-B 라이브 AI 증명 = 포기.** 뒷단 생성은 영상에 안 보임 → 라이브 반증 장치를 새로 만들지 않고 문서로 강조. 단점 감안.

- ✅ **§5-2 산출물 = 데이터 + 서사 둘 다.** AI가 파라미터로 세계를 바꾸고(판정 직결), 신문·지표로 그 변화를 보여준다.
- ✅ **§5-4 주기 = 주(회차) 사이마다.** 매 주 시작에 외생 이벤트 1개, 세계 재구성. N주로 확장.

- ✅ **§5-C 플레이어 매주 행동 = 병원 운영.** 플레이어는 이 세계 속에서 병원을 운영하는 주체다(기존 역할 유지). 세계관 = '실제 세상' — 의료 시스템의 문제는 현실의 법·문화에서 나왔다는 전제이고(§2.1), 그래서 플레이어는 그 현실 속에서 병원을 운영하며 매 주 바뀌는 구조에 부딪힌다.

**남은 세부 (최소 슬라이스에서 확정 — 방향은 안 흔듦)**

- 병원 운영의 구체 범위: 매 주 채용 재편을 어디까지 허용하나(재편 강도)? 이벤트 1종·병원 수 등 최소 슬라이스의 스코프.

## 6. 확정된 통합 그림 — 한 주가 어떻게 도나

여섯 결정을 합치면 한 회차(=한 주)는 이렇게 돈다:

```
주 시작
  │
  ├─ ① 외생 이벤트 1개가 떨어진다 (법 개정 / 수가 인상 / 기사→여론 …)  ← 세계(현실) 자율 변화
  │
  ├─ ② AI가 그 파급으로 세계관을 생성·조절                          ← 새 핵심 축 (뒷단)
  │     · 서사:   법 개정·기사 형태로 사건을 띄운다(이번 주 신문)
  │     · 데이터: 그 뒤에서 병원 N곳의 배후진료·병상·소송리스크를 다시 씀(JSON)
  │
  ├─ ③ 플레이어가 바뀐 세계에서 병원을 운영                         ← '겪는 자' (§5-C 확정)
  │     · 채용·콜 받기/거절 (기존 1막)
  │
  ├─ ④ STEMI 도착 → 전원 뺑뺑이 (버튼식, 입력창 없음)             ← 뺑뺑이 골격 유지
  │     · 바뀐 세계의 병원들을 adjudicate() 결정론 코드가 판정
  │
  └─ ⑤ 주 마감 → 누적 결산 (신문 = 세계 변화 + 돌려보낸 STEMI)
        │
        ▼
   다음 주 → ①로 (N주 확장 시 반복)
```

AI가 만지는 건 오직 **②뿐**이고, ④의 생사 판정은 여전히 코드가 잠근다 — §3 원칙 그대로.

## 7. 다음 단계

1. **남은 열린 지점(§5의 '매주 적응 행동') 확정** → 게임 루프가 닫힌다.
2. 새 축의 **최소 슬라이스** 정의: "외생 이벤트 1종 → AI 세계 재구성 → 신문 고지 + 판정 반영" 한 줄기.
3. TDD로 **"외생 변경 → 세계 파라미터 재구성 → 판정"의 결정론 부분부터** 잠그고, AI 생성은 그 위에 얹는다(기존 2콜 원칙 재사용 — 판정은 코드, 생성은 AI).
4. 확정 후 game-concept §7 · ai-usage-doc §2를 이 축으로 갱신 — 폐기물 정리: 실시간 협상 서술 · 자유 텍스트 입력 UI · 라이브 반증 전략(→ 문서 강조로 교체).
