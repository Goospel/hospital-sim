AI를 쓴 기록이 아니라,
AI를 지휘한 운영체계.

게임 「수화기 너머의 벽」을 만들며, 나는 무엇을 AI에 맡기고 · 어디서 그 권한을 끊고 · 어떻게 강제·기록·학습시킬지를 하나의 운영체계로 짰다. 이 문서는 그 운영체계의 설계도다.

대회는 이렇게 묻는다 — "AI를 활용하는 사람은 많다. 우리가 찾는 건 AI의 다음 단계를 설계할 디렉터다."
그래서 자랑거리는 "AI를 썼다"가 아니라, 규칙 → 강제 → 지휘 → 학습으로 이어지는 지휘 체계 그 자체다.

    §2 글로벌 규칙 · 루트 CLAUDE.md

    모든 세션에 자동 로드되는 나의 메타 시스템.

    프로젝트를 가리지 않고 모든 Claude Code 세션에 로드되는 개인 규칙. 이건 특정 코드가 아니라 "AI와 일하는 방식" 자체를 못박은 계약이다. 핵심 여덟 갈래:

    언어

    사용자 대면 텍스트는 항상 한국어

    일반 답변뿐 아니라 질문 옵션·계획 요약·PR/커밋 요약까지. 영어 툴 출력에 끌려가는 드리프트는 훅(§4)이 매 턴 되잡는다.

    Git

    main 직접 작업 금지 → 브랜치 → PR → 사용자에게 머지 물음 → 승인 후 머지

    선제 머지 금지. 워크트리 분리 전 로컬 main을 먼저 최신화(stale base 금지). squash + 브랜치 삭제. 이 문서도 그 절차로 올라간다.

    TDD

    Red → Green → Refactor — 실패를 눈으로 본 뒤 구현

    기능은 실패하는 테스트(엣지·에러 포함)부터. 게임의 논지를 지탱하는 불변식을 LLM을 붙이기 전에 코드로 잠근다.

    추적

    작업 추적 3종 — plan · changeLog · troubleshooting

    매 작업(대체로 PR)마다 한 세트로 갱신. "무엇을 / 왜 / 어떤 함정"을 이력으로 남겨 AI의 작업을 감사 가능하게 만든다. (이 프로젝트는 claude-docs/에 격리.)

    PKM

    학습 파이프라인 — 현장 메모 → 정제 공개 → 사적 보관

    learning-notes(현장) → goospel.github.io(일반화·공개) → learning-vault(사적). "같은 질문 두 번 안 묻기"가 원칙.

    메모리

    자동 메모리 위생 — 인덱스 자동생성 + 정기 정리

    세션 간 사실 파일 + 자동 재생성 인덱스(§8). 단일 출처 = 각 메모 frontmatter → 인덱스 drift 구조적 차단.

    인코딩

    Windows 셸 한글·인코딩 원칙 (무성 실패 3종 방어)

    ① Git Bash grep/sed 멀티바이트 무성실패 → ripgrep/PowerShell 리터럴. ② PS 5.1 인라인 한글 CP949 깨짐 → 파일 경유(git commit -F). ③ 파일 재생성 시 BOM·EOL 보존.

    왜 이게 디렉팅인가 규칙을 세션마다 다시 설명하지 않는다. 한 번 못박아 모든 세션이 같은 헌법 위에서 돌게 만든 것 — 이게 개별 지시가 아니라 시스템을 설계한다는 뜻이다.
    §3 프로젝트 전용 규칙

    이 저장소에만 걸리는 헌법 — 그리고 게임의 심장.

    글로벌 규칙 위에, 이 프로젝트만의 제약을 얹는다. 그중 하나는 게임의 메시지를 지키는 아키텍처 결정 그 자체다.

    AGENTS.md — 프로젝트 헌법
    # This is NOT the Next.js you know

이 버전은 breaking change가 있다 — API·규칙·파일구조가
학습 데이터와 다를 수 있다. 코드 작성 전에
node_modules/next/dist/docs/ 의 가이드를 먼저 읽어라.

    AI가 낡은 지식으로 코드 쓰는 걸 막는 하드가드

    LLM은 학습 시점 이후 바뀐 프레임워크를 모른다. 그래서 "네 학습지식을 믿지 말고 이 레포의 실제 문서를 먼저 읽어라"를 프로젝트 헌법에 못박았다. CLAUDE.md는 이 AGENTS.md를 @import한다.

    이건 본선 제출물인 "AI 에이전트 설계서 / 디렉팅 명세서"의 사전 과제판 — 무엇을 시키고, 무엇을 금지할지 미리 못박은 명세다.

    디렉팅의 크라운 주얼 — 판정 = 코드 / 대사 = LLM

    전원 담당자를 LLM에 그냥 맡기면, 플레이어가 절박하게 매달릴수록 LLM이 동조해 "…받을게요"로 굴복한다 — 그 순간 "구조의 벽은 안 뚫린다"는 게임의 메시지가 정반대로 뒤집힌다. 그래서 권한을 타입 수준에서 갈랐다.

    ● 판정콜 — 결정론 코드

    상태 전이의 유일한 권한

    수용/거절은 병원의 숨은 제약(병상·당직·배후진료)만 읽는 순수 함수가 확정한다. 설득 텍스트를 받는 파라미터가 함수 시그니처에 아예 없다 — "매달리면 뚫린다"는 붕괴가 런타임 방어가 아니라 타입상 표현 불가능.

    Type-level wall
    ● 대사콜 — LLM

    확정된 결과를 연기만

    이미 정해진 판정을 인물 대사로 옮길 뿐. 매달릴수록 담당자는 냉정해지되(에스컬레이션) 판정은 절대 안 바뀐다. 무키·타임아웃 시 결정론 폴백으로 자동 강등 — API 키 없이도 완주.

    src/game/adjudicate.ts — 2콜의 "판정콜"
    // 입력은 병원의 숨은 제약과 환자뿐. 설득 텍스트는 이 함수에 타입으로 도달할 수 없다.
export function adjudicateTransfer(hospital: Hospital, patient: Patient): TransferVerdict {
  if (hospital.beds <= 0)          return { accepted: false, reason: 'NO_BED' }
  if (!hospital.hasErOnCall)       return { accepted: false, reason: 'NO_ER_ONCALL' }
  if (hospital.overcrowded)        return { accepted: false, reason: 'ER_OVERCROWDED' }
  if (!hospital.backupCare.includes(patient.requiredSpecialty))
                                   return { accepted: false, reason: 'NO_BACKUP_CARE' }
  return { accepted: true }
}

(K @(0xBC18,0xB4DC,0xC2DC))  # "반드시"
(K @(0xC790,0xB3D9,0xC73C,0xB85C)) # "자동으로"

    §7 프롬프트 ↔ 실행 로그

    디렉터의 프롬프트는 짧다.

    한 줄 지시가 운영체계(규칙·훅·스킬)를 타고 여러 AI의 다단계 작업으로 펼쳐진다. 지시가 짧을 수 있는 건, 나머지를 시스템이 감당하도록 미리 설계했기 때문이다. 아래는 실제 지휘 프롬프트와 그 결과다.

    §8 메모리 · 세션을 넘는 학습

    배운 걸 다음 세션의 Claude가 안다.

    ~/.claude/…/memory/에 사실을 한 파일씩 쌓고, 인덱스는 훅이 각 파일 frontmatter에서 자동 재생성한다. 본문 하나만 고치면 인덱스가 따라오므로 "본문↔인덱스" drift가 구조적으로 불가능하다.

    §9 되먹임 사다리 · 시스템이 스스로 굳는다

    실수는 한 번만 겪는다 — 그다음엔 규칙이 된다.

    함정을 만나면 그 자리에서 끝내지 않고, 재발 강도에 따라 위로 승격시킨다. 아래로 갈수록 강제력이 세진다.

    ① 함정 기록
    1분+ 디버깅한 함정은 troubleshooting.md에 T-### 한 항목(증상/원인/해결/재발방지).
    ② 메모리 승격
    세션을 넘겨야 할 교훈은 memory 사실 파일로 (예: tsc 게이트 필수).
    ③ 글로벌 원칙
    다른 맥락에서 2회+ 재발하면 루트 CLAUDE.md 원칙으로 (예: Windows 인코딩 3원칙).
    ④ 하드 훅
    사람 판단에 맡기면 안 되는 건 settings.json 훅으로 실행 강제 (예: 언어 리마인더).
    이 사다리가 곧 "AI의 다음 단계 설계" 디렉터의 일은 완벽한 첫 지시가 아니라, 겪은 걸 시스템에 되먹여 다음번엔 안 겪게 만드는 것이다. 이 운영체계는 매 실수마다 조금씩 더 단단해진다.
    디렉터의 원칙 · 한 줄로
    LLM을 루프에 넣는 건 누구나 한다 — 요점은 넣은 뒤 어디서 끊었는가다.
    상태를 바꾸는 판정은 언제나 코드가. AI는 그 결과를 연기하거나 설명할 뿐.
    규칙은 세션마다 설명하지 말고 한 번 못박아 시스템으로.
    실수는 troubleshooting → 메모리 → 원칙 → 훅으로 승격시켜 다신 안 겪는다.
