## 🔑 선결 — 오늘(7/16) **사용자가** 확인/결정할 3가지

코드가 아니라 규정·계정 문제라 내가 대신 못 한다. 이게 막히면 뒤 일정이 통째로 흔들리니 **가장 먼저**.

1. **제출 요강/폼 전문 정독 → 필수항목·마감 시각 확정.** 특히
   - (a) 빌드 제출 형식: "라이브 Vercel URL만" 허용인가, 아니면 **다운로드 가능한 웹빌드 아티팩트(zip 등)도 별도 요구**인가?
   - (b) **마감 정확 시각·타임존** (8/10 00:00 vs 23:59 KST?).
   - (c) PDF 규격/용량(폼상 1GB 확인됨).
   → 폼을 네가 갖고 있으니 같이 확정하자.
2. **Anthropic 콘솔에서 결제/크레딧 활성 상태 직접 확인.** 결제 설정은 내가 대행 불가(금지 영역) → 네가 직접. 미설정이면 실LLM(X8)은 **처음부터 스킵 전제**로 잡고 ④PDF를 "설계 + 로컬 시연" 중심으로 쓴다.
3. **대회 FAQ '유료 API 비용 부담 주체' 문의 발송 여부 결정.** 초안은 내가 준비하되, **발송은 네 승인 후**.

## 최소 제출 가능선 (min cut line) — 목표 **8/1 (U5)**

LLM/유료 API가 **전혀 없어도**(결정론 폴백 대사) Vercel 공개 URL에서 STEMI 한 판이 마우스만으로 처음부터 끝까지 완주되고, 그 한 판이 데모 4비트를 화면에서 모두 보여주는 상태:

1. 응급환자 도착 + 실시간 골든타임 카운트다운
2. 전원 콜 반복 → `NO_BED`/`NO_SPECIALIST` 거절 사유 태그 누적
3. 자유 텍스트 설득 입력이 있으나 **'병상 0'은 어떤 문장으로도 안 뚫림**(하드락 벽 가시화)
4. 결말(수용 or 골든타임 소진 사망) → 인과 디브리핑(구조 변수 3개 + 반사실 1쌍 + "당신 판단은 옳았다, 문제는 구조였다") + README 배포 링크·실행법

이 선이 서면 ①빌드+소스가 완성되고, 이 화면으로 ②영상·③④PDF 집필이 모두 가능해져 **4종 전부 제출 가능**.

## LLM 삽입 지점 (판정=코드 / 대사=LLM)

최소선(U5) 확보 후 `X8`에서 얹는다. 판정은 코드가 이미 확정, **LLM은 그 결과를 인물 대사로 연기만** → 게임 상태 전이는 LLM 손에 안 넘어감(정합성 테스트로 타입·런타임 양쪽 잠금). 무키·API 실패·상한 초과 시 결정론 폴백으로 **자동 강등**. 기존 폴백 경로는 삭제하지 않고 그 위에 옵셔널 레이어로만 끼운다 → LLM은 전량 upside, 실패해도 되돌릴 최소선이 항상 손에 있다. **이 강등 구조 자체가 ④AI PDF의 셀링**(설득으로 병상0 벽을 못 뚫는 붕괴를 타입 수준에서 차단)이 된다.
