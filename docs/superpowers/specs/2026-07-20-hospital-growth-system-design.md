# 설계 — 병원 성장 시스템 (재투자 루프 + 제로섬 카운터)

> **한 줄**: 매주 번 돈으로 병원을 키우되, 커져도(또는 선의로 적자를 감수해도) 시스템 문제가 안 풀리는 걸 **값으로** 겪게 한다.
> **근거**: [hospital-growth-dilemma-grounding.md](../../research/hospital-growth-dilemma-grounding.md)(4축 실측 검증) · [fee-schedule-and-subsidies.md](../../research/fee-schedule-and-subsidies.md)(교차보조·재정중립).
> **관련**: [game-concept.md §3](../../concept/game-concept.md) · [changeLog.md](../../../claude-docs/changeLog.md) · plan.md의 ⏸ '재투자·조작 루프(U4.5 B안)'의 실현.

---

## 1. 문제 · 목적

**현 공백**: 개원 위저드(100억)는 1회뿐이고 이후 `hospital`은 동결이다. 매주 번 돈은 `cumulativeNetBillions`에 쌓이기만 하고 **쓸 데가 없다** — "마냥 돈만 번다"(사용자).

**목적**: 주 사이에 **재투자 루프**를 넣어 번 돈으로 병원을 키우되, 세 딜레마가 성장을 따라오게 한다 — 커진다고 응급실 뺑뺑이·필수의료 붕괴·지방 공백이 사라지지 않는다. 특히 **"수익 최소화(배후과만)했더니 내 병원은 조용해지는데, 지속도 안 되고(②) 전국·지방은 그대로(①③)"**라는 두 겹의 catch를 값으로 실현한다(사용자 의도).

**설계 원칙(기존 헌법 계승)**: 성장은 **채용 경제·용량만** 바꾼다. `adjudicateTransfer` 판정 규칙은 한 줄도 안 건드린다(세계 이벤트 헌법과 동일). show-don't-tell — 해석 카피 0, 숫자만.

## 2. 범위 (확정 절단선)

**코어 (이 스펙)**:
- **레버 A — 채용 재투자**: 금고 예산으로 개원 위저드를 재실행(증축만, 해고 없음).
- **레버 B — 병상 증설**: 병상 티어↑ = 과당 채용 상한↑ + 하루 볼륨↑. 지역 총량 규제로 비용/상한.
- **② 금고**: 누적 순이익이 쓸 수 있는 잔고가 되고, 배후과 적자가 금고를 말린다.
- **①③ 시스템 카운터**: 전국 배후과 풀(소진 게이트 + 배경 감소) + 에필로그 전국·지방 신문 병치.

**비범위 (후속 — 근거는 보존)**:
- **상급종합 지위 상승**(중증 34%·소아/산부 상시입원 강제 = ④+② 정점). 근거 [grounding §④]. 코어는 병상 총량 규제만 얇게.
- 진짜 지도·거리 시뮬(③의 공간화) — 리서치가 "정밀 인과 실증 없음"이라 추상 카운터가 오히려 정직.
- 간호·병상 세부, 분원(별도 병원) 개설, 다병원 지역 집계.

## 3. 아키텍처 개요

```
LANDING → WORLD_EVENT → SETUP → (RECEIVING → DAY_END)×7 → WEEK_SUMMARY
  WEEK_SUMMARY ─[재투자·다음 주]→ WORLD_EVENT(공문) → GROWTH(재투자) → beginWeek → RECEIVING …
              └[종료]────────────────────────────────────────────────→ EPILOGUE
```
- 새 페이즈 **GROWTH** 하나. 새 순수 모듈 **`growth.ts`**(금고·성장 적용) + **`system.ts`**(전국 풀). 나머지는 기존 위저드(`setup.ts`)·신문(`news.ts`)·장부(`ledger.ts`) 재사용.
- 유일한 새 서브시스템 = **전국 배후과 풀**. 그 외는 기존 배관 위 가산 레이어.

## 4. 상태 모델 (`types.ts` · `session.ts`)

`SessionState`에 추가:
```ts
choices: SetupChoices   // 현재 병원 명단(매주 성장). completeSetup이 보존(지금은 버림).
beds: number            // 병상 티어. 초기 FIXED_BEDS(3). 성장으로 상승.
treasury: number        // 금고 잔고(억). 쓸 수 있는 돈. (②)
system: SystemState     // 전국 배후과 풀. (①③)
```
새 타입:
```ts
interface SystemState {
  pool: Record<Specialty, number>         // 전국 배후과 잔여 풀(게임 스케일 각색)
  poolInitial: Record<Specialty, number>  // 표시용 초기값(에필로그 "N → 잔여")
}
```
- `hospital.beds`·`MAX_DOCTORS_PER_DEPT`는 상수에서 **세션 파생값**으로. `FIXED_BEDS`는 초기값으로 남기고, 실효 상한 = `state.beds`.
- **불변·결정론 유지**: 모든 성장 함수는 순수. RNG 0(배경 감소도 seed 파생).

## 5. 페이즈 전이 (`session.ts`)

- `completeSetup(choices, world)`: 기존 + `choices` 보존, `beds = FIXED_BEDS`, `treasury = SETUP_BUDGET_BILLIONS − hiringCost(choices)`(개원 잔액 = **기본안**, §13에서 '0 시작' 대안과 함께 확정), `system = initSystem()`.
- `completeWeek`: 기존 + `treasury += weekNet`(이번 주 `ledgerDays` 순이익 합). 금고가 매주 영업손익을 흡수 — 배후과 많으면 준다(②).
- `nextWeek`: 기존(WORLD_EVENT로) — 여기서 **배경 풀 감소**를 적용(다른 병원·은퇴로 전국 풀이 매주 줄어든다, seed 결정론).
- **새 함수** `applyGrowth(state, nextChoices, nextBeds): SessionState`:
  - 검증: `nextChoices ≥ choices`(해고 없음), `withinTreasury`(델타 비용 ≤ treasury), `withinDeptCaps(nextBeds)`, 그리고 **풀 잔여** 이내(배후과 증분 ≤ `system.pool`).
  - 적용: `hospital = buildHospital(nextChoices, world.departments)` 재구성(+ beds 반영), `treasury −= 델타 비용`, `system.pool −= 배후과 증분`, `choices = nextChoices`, `beds = nextBeds`.
  - GROWTH → beginWeek로 이어진다(같은 병원, 새 주 1일차).
- WEEK_SUMMARY의 `onNextWeek`가 GROWTH 경로를 열고, WorldEventCard(2주차+) `onContinue`가 `enterGrowth`로.

## 6. 레버 A — 채용 재투자 (`setup.ts` 재사용)

GROWTH 화면 = **개원 위저드 재실행**. `SetupWizard`를 예산=`treasury`, 시작 명단=`choices`, 하한=현재 명단(증축만)으로 연다.
- 배후과를 넓히면 `NO_BACKUP_CARE`/`NO_NIGHT_BACKUP`이 준다(24h는 과당 2명, `ROUND_THE_CLOCK_MIN_DOCTORS`).
- 수익과(미용·검진)는 금고를 채우되 응급엔 무력 — 성장 자금원이지 해결책이 아니다.
- **깊이(과당 3번째)의 한계**: 하루 볼륨이 낮으면 같은 과 동시 부하가 2를 넘는 날이 드물어 3번째 의사가 거의 안 쓰인다(`setup.ts` 기존 주석). → 깊이가 의미를 가지려면 **레버 B(볼륨)**가 필요. 이 결합이 "더 키우면 더 뽑아야 한다"를 만든다.

기존 `adjustDoctors`·`hiringCost`·`withinBudget`·`buildHospital` 그대로. 예산 인자만 `treasury`로.

## 7. 레버 B — 병상 증설 (`setup.ts`·`daysim.ts`·`receiving.ts`)

병상은 판정에서 죽은 필드(`adjudicate.ts`: "beds 안 본다")라, **성장의 실감**을 주려면 볼륨을 연동한다:
- **병상 티어**(예: 3 → 5 → 7, 튜닝값): `MAX_DOCTORS_PER_DEPT = beds`(과당 상한↑) + **하루 콜 수 = 티어별 하드코딩 상수**(예: 3병상=5통·5병상=7통·7병상=9통, 결정론·튜닝값). 병상을 키우면 **환자도 그만큼 더 온다**(큰 병원 = 더 많은 환자, 사용자 결정) — 안 그러면 늘린 병상에 받을 사람이 없어 병상이 무의미.
- **콜 볼륨 스케일**: `createCallQueue(week, day, beds)`가 티어별 콜 수만큼 결정론적으로 큐를 만든다(기본 `DAY_PLANS` 연장 규칙: 추가분은 뒤로 갈수록 필수 응급↑ 원칙 유지, 라벨은 kind 풀 순번 파생 — 새 콘텐츠 0).
- **진실된 의미(리서치 핵심)**: 병상만 늘리고 배후과를 안 채우면 **자리는 늘었는데 여전히 뺑뺑이**(뺑뺑이 = 배후 불가지 병상 아님). 판정은 여전히 배후·점유가 정한다.
- **④ 지역 총량 규제**: 병상 증설 비용이 티어마다 오르거나 상한이 걸린다(공급 제한/조정 지역 각색). 코어는 **비용 체증 + 하드 상한** 한 줄로.

**⚠️ 최대 구현 리스크 — I8 재유도**: 콜 볼륨이 5 고정에서 가변이 되면 (1) 야간 판정은 `arrivalMinFor(...,count)`가 이미 count 파생이라 자동 정합, (2) 그러나 불변식 **I8(|주간 순이익| ≤ 4 × 예산)**은 콜 델타 누적이 늘어 깨질 수 있고 **테스트로 안 잡힌다**. → 최대 병상에서 **브라우저 7일 완주 + 콘솔 0**으로 잠근다(기존 규율). 콜 델타·볼륨 상한을 이 체크로 튜닝.

## 8. 딜레마 층 ② — 금고 (재정 지속불가)

- `treasury` = 개원 잔액에서 출발, 매주 `+= weekNet`, 성장에서 `−= 델타 비용`.
- 배후과는 구조적 적자(`DEPARTMENTS` 음수 = 24h 대기 고정비)라, **배후과만 채우면 금고가 말라** 다음 성장·유지가 불가 → 미용을 뽑아야 지속(피하려던 벽 복귀). 근거: 원가보전율 100% 미달·교차보조([grounding §②]).
- 금고 음수 허용(영업 적자 누적) → 그 주엔 성장 0으로 넘어간다(성장 스킵은 가능, 강제 아님). "선의의 파산"이 값으로.
- WEEK_SUMMARY·GROWTH 화면에 **금고 잔고**를 노출(쓸 수 있는 단일 숫자). 기존 `cumulativeNetBillions`(영업 누적)와 별개 라인.

## 9. 딜레마 층 ①③ — 시스템 카운터 (`system.ts`)

**핵심: ①과 ③은 한 풀의 두 라벨.** ① "전국 풀에서 뺀다" = ③ "그 인력이 지방에서 빠진다".

- `system.pool[S]` = **이 게임 세상에 존재하는(아직 배치 안 된) 그 과 의사의 수 = 하드코딩된 유한값**(사용자 결정). "의사를 뽑고 싶어도 못 뽑는" 상황의 구현체다 — 세상에 의사가 몇 명인지가 고정돼 있어, 그 유한 공급이 채용의 진짜 천장이다. 초기값은 게임 스케일 각색(부호·소규모·재생 불가만 근거 — [grounding §① 왜곡 금지] 준수: 실제 헤드카운트를 '현재 실측'으로 표기하지 않고 게임 세계 각색값으로).
- **채용 = 이동, 창출 아님**: 배후과 N명 채용 → `pool[S] −= N`. GROWTH 화면이 잔여를 표시("전국 순환기 배후 잔여 M"). 돈은 있어도 세상에 없는 의사는 못 산다.
- **소진 = 하드 게이트**: `pool[S] = 0`이면 그 과는 **돈이 있어도 못 뽑는다**("전국에 남은 전문의 없음"). 병상0 하드락과 같은 철학 — 돈으로 못 사는 벽. 이게 ①의 가장 날카로운 교훈.
- **배경 감소**: 매주(`nextWeek`) `pool[S] −= 소량`(다른 병원 채용·은퇴>배출, seed 결정론). 내가 안 뽑아도 풀이 준다 → **선의의 플레이어도 전국 예비가 마르는 걸 본다**(③ '지방 비어감' 심화). 근거: 흉부외과 배출<은퇴 역전([grounding §①]). 감소량은 튜닝값.
- **⚠️ 왜곡 금지 준수**: "내 성장 X → 지방 Y 감소"의 **정밀 결합계수는 단정 안 함**. 풀 차감은 **방향**만(1명 뽑으면 1명 빠짐이라는 회계적 항등이지 지역 인과 측정 아님). 전국 신문은 **고정 실제 앵커**만 배경으로.

## 10. 에필로그 병치 (`Epilogue.tsx`·`ledger.ts`)

기존 `내 신문(돌려보낸 응급) ↔ 병원 장부` 병치에 **전국·지방 층**을 더한다:
- **내 병원**(게임 산출, 성장으로 개선 가능): 이번 판 받은/돌려보낸 응급, 금고 결산.
- **전국·지방**(고정 실제 앵커, 안 줄어듦): 강원 산부인과 입원 **40.3% 수도권 유출** · 수도권 분원 **6,600병상** · 재이송 **5,657건**(2024). + 전국 풀 소진("배후과 예비 N → 잔여 M").
- 병치가 논지다(카피 0): 내 신문은 얇아져도 전국·지방은 그대로 → **"국지적 승리는 확장 안 되고, 내가 채운 자리는 어딘가에서 빠진 자리."**

## 11. 불변식 · 헌법

- **판정 불가침**: `adjudicateTransfer`·`hardlockReason` 로직 무변경. 성장은 `hospital`의 `backupCare`/`roundTheClockBackup`/`roster`/`beds`와 경제만 바꾼다(위저드가 이미 하던 재구성).
- **I8**: |주간 순이익| ≤ 4 × 예산 — 병상 볼륨 스케일 후 최대 병상에서 브라우저로 재확인.
- **결정론**: 성장 적용·풀 감소·볼륨 연장 전부 순수·seed 파생. 같은 선택 = 같은 결과.
- **show-don't-tell**: 성장·에필로그 화면에 해석 카피 0. 금고·비용·풀 잔여·두 신문의 숫자만.
- **왜곡 금지([grounding])**: 앵커는 2018 라벨·급여 한정·국고보조 후·3센터 표본·비중≠절대수·정밀 결합계수 금지를 지킨다. 반증 3건(흉부 629 집중·병상 사전승인 절차·외상 67.17%)은 안 쓴다.

## 12. 테스트 전략 (TDD)

Red→Green + `tsc --noEmit` 게이트(vitest는 타입 미검사, 회귀 위험). 순수 코어부터:
1. **금고 산수**(`growth.ts`): 개원 잔액·주간 누적·성장 차감·음수 허용·`withinTreasury`.
2. **성장 적용**(`applyGrowth`): 해고 방지(하한)·병원 재구성 정합·delta 비용·choices 보존.
3. **전국 풀**(`system.ts`): 채용 차감·소진 하드 게이트(0이면 불가)·배경 감소 결정론.
4. **병상 볼륨**(`createCallQueue(beds)`): beds별 콜 수·야간 파생 정합·id 고유성·결정론.
5. **전이**(`session.ts`): completeSetup 보존·completeWeek 금고 누적·nextWeek 배경 감소·applyGrowth→beginWeek.
6. **회귀**: 기존 302 vitest green 유지(성장 미사용 경로 = 기존 동작 불변, 예: beds 기본 3·choices 기본).
7. **브라우저 I8**: 최대 병상 7일 완주 + 콘솔 0(테스트 밖 유일 검증).

## 13. 미해결 — 계획(writing-plans)에서 결정할 구현 질문

- **콜 볼륨 티어 수치**(메커니즘은 확정 = 티어별 하드코딩): 티어별 콜 수 상수(예: 5/7/9), 추가분 kind 배치(필수 응급↑ 원칙 유지)의 결정론 규칙.
- **전국 풀 초기값·배경 감소량**: 세상에 존재하는 과별 의사 수(게임 스케일 각색)와 주당 감소량 — I8·플레이 길이와 함께 튜닝.
- **병상 티어·비용 곡선**: 티어 수, 티어당 비용 체증, 하드 상한(④ 지역 규제 각색).
- **금고 초기값**: 개원 잔액 시작 vs 0 시작(1주차 성장 가능 여부에 영향).
- **GROWTH UI**: SetupWizard 재사용 정도(그대로 vs 성장 전용 래퍼 — 병상 컨트롤·풀 잔여·금고 표시 추가).
