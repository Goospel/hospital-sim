# 설계 스펙 — 세계 이벤트 용어 풀이 (opt-in 용어집)

> **문제**: 세계 이벤트 헤드라인(예: "필수의료 정책수가 인상 의결 — 재정중립 원칙 따라 타 급여수가 동반 조정")은 전문용어 덩어리라, 일반 플레이어는 무슨 말인지 모른다. 알고 싶은 사람에게만 뜻을 알려주는 편의 기능이 필요하다.
> **원칙 충돌 해소**: 이 게임의 핵심은 show-don't-tell(해석 카피 금지 — 숫자로 스스로 깨닫게)이다. 그래서 (1) **기본 접힘·opt-in**으로 안 보고 싶은 사람의 경험을 불변으로 두고, (2) 설명 내용은 **용어 정의 + 현실 맥락(사실)** 까지만 — "이 뉴스가 네 병원에 뭘 하는지"는 절대 짚지 않는다. 그 낙차는 여전히 플레이어가 장부에서 발견한다.
> **근거**: [fee-schedule-and-subsidies.md](../../research/fee-schedule-and-subsidies.md) · [essential-care-economics.md](../../research/essential-care-economics.md) · [medical-system-grounding.md](../../research/medical-system-grounding.md).

## 1. 확정된 설계 결정 (브레인스토밍)

| 축 | 결정 | 이유 |
|---|---|---|
| **설명 깊이** | 용어 정의 + 현실 맥락 (게임 내 효과는 **안** 짚음) | 사실 근거·다큐 톤에 맞고, show-don't-tell 유지 |
| **범위** | 세계 이벤트 화면만 (컴포넌트·데이터는 재사용 가능하게) | 사용자가 지적한 지점부터. 응급 접수 등 확장은 뒤로 |
| **인터랙션** | 헤드라인 아래 접기/펼치기 토글 | 미니멀 카드 유지·기본 접힘으로 방해 0. 헤드라인이 LLM 슬롯이라 문자열 포함 감지가 위치 마킹보다 강건 |

## 2. 아키텍처

세 조각. 데이터(순수)·표시(컴포넌트)·배선을 분리한다.

```
src/game/glossary.ts        ← 순수 데이터 + 감지 셀렉터 (테스트 대상)
src/components/TermGlossary.tsx  ← 접기/펼치기 UI (재사용 가능)
src/components/WorldEventCard.tsx ← 헤드라인 아래 한 줄 배선
```

### 2.1 데이터 — `src/game/glossary.ts`

```ts
export interface GlossaryEntry {
  term: string        // 대표 표기(펼침 패널 제목)
  match: string[]     // 헤드라인에서 감지할 문자열들(term + 별칭)
  def: string         // 한 줄 중립 정의
  context: string     // 한 줄 현실 맥락(사실) — 게임 효과는 X
}

export const GLOSSARY: GlossaryEntry[]         // 아래 §4 콘텐츠
export function termsInText(text: string): GlossaryEntry[]
```

`termsInText` — **긴 문자열 우선·비중복(masking) 매칭**:

1. 모든 `(matchString, entry)` 쌍을 `matchString` 길이 내림차순 정렬.
2. `text`를 스캔하며 각 matchString의 등장 위치를 찾되, **이미 claim된 문자 구간과 겹치면 건너뛴다**. 매칭되면 그 구간을 claim하고 `{entry, index}` 기록.
3. entry 기준 dedupe(한 entry가 여러 번/여러 별칭으로 걸리면 **가장 이른 index** 유지).
4. index 오름차순 정렬 → **헤드라인 읽는 순서**로 반환.

> **왜 masking인가**: `급여 ⊂ 비급여`, `수가 ⊂ 정책수가`. 긴 것을 먼저 claim해 마스킹하지 않으면 "비급여"에서 "급여"가, "정책수가"에서 "수가"가 오탐된다. 이 게임은 메시지 정확도가 핵심이라(메모 fact-grounding) 오탐은 곧 거짓말이다.

### 2.2 컴포넌트 — `src/components/TermGlossary.tsx`

- props: `{ text: string }`.
- 내부: `const entries = termsInText(text)`. **`entries.length === 0`이면 `null` 반환**(아무것도 안 그림 — 아는 용어가 없는 헤드라인엔 토글조차 안 뜬다).
- `useState(false)`로 접힘/펼침. 접힘: 은은한 버튼 `용어 풀이 ▾`(muted, `text-xs`). 펼침: `▴` + 리스트, 각 entry = **`term`(굵게)** — `def` / `context`(더 흐리게, 작게).
- 접근성: `<button aria-expanded>` + 패널 `id`/`aria-controls`. 각 entry는 `<dl>`/`<dt>`/`<dd>` 시맨틱.

### 2.3 배선 — `src/components/WorldEventCard.tsx`

`<header>` 바로 아래 `<TermGlossary text={event.headline} />` 한 줄. 카드는 접힘 상태에선 작은 토글만 늘어 미니멀함 유지.

## 3. 테스트 (TDD — RED 먼저)

`src/game/glossary.test.ts` (vitest):

- **감지**: `"필수의료 정책수가 인상 의결 — 재정중립 원칙 따라 타 급여수가 동반 조정"` → `[필수의료, 수가, 재정중립, 급여/비급여]` (읽는 순서).
- **미포함 제외**: 아는 용어 없는 문자열 → `[]`.
- **비중복 오탐 방지 ①**: `"비급여"` 포함 → 급여/비급여 entry 1건, 안쪽 `급여` 중복 아님.
- **비중복 오탐 방지 ②**: `"정책수가"` 포함 → 수가 entry 1건, 안쪽 `수가` 중복 아님.
- **분해**: `"급여수가"` → `[급여/비급여, 수가]` (급여가 앞이라 급여 먼저). `수가` 별칭에 `급여수가`를 넣지 않아 급여+수가 둘로 자연 분해된다.
- **중복 제거**: 같은 용어 2회 등장 → entry 1건.
- **순서**: 뒤에 오는 용어가 뒤에.

컴포넌트(`TermGlossary`)는 로직이 얇아(토글 state뿐) 단위 테스트 생략 — 브라우저 실측(펼침/접힘·용어 렌더·용어 0 헤드라인엔 미표시)으로 검증. `tsc --noEmit` + `vitest run` 게이트(메모 vitest-no-typecheck-tsc-gate).

## 4. 콘텐츠 — 시드 용어집 (사실 근거)

> 각 정의는 리서치 문서 근거. **"과별 수가 인상" 같은 표현 금지**(fee-schedule §1 경고: 수가는 과가 아니라 '행위'에 붙는다). 게임 내 효과·해석은 넣지 않는다.

| term | match(별칭) | def(한 줄) | context(현실 맥락) |
|---|---|---|---|
| **수가** | 수가, 정책수가 | 건강보험이 진료 행위마다 정하는 공정 가격(상대가치점수×환산지수×종별가산율, 건정심 의결). | 병원이 아니라 정부가 정한다. 원가에 못 미치게 책정된 행위는 볼수록 적자. '정책수가'는 그중 필수의료 행위에 얹는 별도 보상. |
| **재정중립** | 재정중립 | 건강보험 총지출을 그대로 두는 원칙 — 한 항목 수가를 올리면 다른 항목을 그만큼 내린다. | 총액이 고정이라 '필수의료 수가 인상'이 다른 급여 진료 인하로 상쇄될 수 있다(악의가 아니라 산수). |
| **급여 / 비급여** | 비급여, 급여 | 급여=건강보험이 값을 정하고 일부를 대주는 진료. 비급여=보험 밖이라 병원이 가격을 자율로 정하는 진료(미용 등). | 급여(특히 필수의료)는 고시가가 원가에 못 미치기 쉽고, 병원은 그 손실을 비급여 수익으로 메워왔다. (비급여도 범위·공시는 규제받는다.) |
| **필수의료** | 필수의료 | 생명에 직결되지만 저수가·고위험이라 병원이 기피하기 쉬운 분야(응급·외상·심뇌혈관·분만·소아 등). | 낮은 보상과 높은 소송 위험으로 인력이 빠져 '응급실 뺑뺑이'의 구조적 배경이 된다. |
| **배후진료** | 배후진료, 배후 | 응급 환자를 실제로 수술·시술할 전문과(순환기내과·흉부외과 등)가 병원 뒤에서 대기하는 것. | 배후과가 없으면 응급실 문이 열려 있어도 그 환자를 받을 수 없다. |

> `match` 배열은 **긴 별칭이 먼저 오도록** 넣지 않아도 된다 — `termsInText`가 전체를 길이순 정렬하므로. `급여/비급여`는 `비급여`가 `급여`보다 길어 masking이 자연히 처리한다. **`수가`에 `급여수가`를 넣지 않은 이유**: 그러면 헤드라인의 "급여수가"가 통째로 수가에 먹혀 `급여/비급여`가 안 뜬다 → 플레이어에겐 둘 다 유용하므로 급여+수가로 분해되게 둔다(§3 분해 테스트).

## 5. 범위 밖 (나중에)

- 응급 접수(원가·수가·급여) 등 게임 전체 배선 — `TermGlossary`·`GLOSSARY`는 재사용 가능하게 두되 이번엔 배선 안 함.
- 헤드라인 내 용어 위치 마킹/툴팁(접근안 B) — LLM 슬롯 취약성으로 기각.
- 별도 용어집 모달(접근안 C) — 범위 과함으로 기각.

## 6. show-don't-tell 자기검증

- 기본 접힘 → 안 펼친 플레이어의 화면은 지금과 **바이트 단위로 동일**.
- 콘텐츠에 "그래서 이게 불공정하다/너에게 손해다" 류 **평가·게임효과 문장 0**. 정의와 현실 사실만.
- 아는 용어 0 헤드라인엔 토글도 안 뜸 → 미니멀함 보존.
