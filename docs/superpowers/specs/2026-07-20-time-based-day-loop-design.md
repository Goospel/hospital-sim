# 설계 — 시간 기반 하루 루프 · 전문의 점유 벽

> **한 줄**: 하루를 "콜 5통 고정 큐"에서 **9시~마감 시계**로 바꾼다. 진료마다 seed 결정론 소요시간을 줘 의사가 그 시간만큼 점유되고, 응급이 왔을 때 그 배후과 전문의가 **돈 되는 예약진료로 점유돼 자유롭지 않으면 벽**이 된다. 거절 버튼은 사라지고, 응급 못 받음은 능동 행위가 아니라 구조적 귀결이 된다.
>
> **관련**: [physician-workflow-and-backup-occupancy.md](../../research/physician-workflow-and-backup-occupancy.md)(전제 팩트체크) · [medical-system-grounding.md](../../research/medical-system-grounding.md)(뺑뺑이 1위=배후진료 부재) · [fee-schedule-and-subsidies.md](../../research/fee-schedule-and-subsidies.md)(수가 부호) · [2026-07-20-doctor-roster-visualization-design.md](2026-07-20-doctor-roster-visualization-design.md)(의사 유닛·피로 — 이 위에 얹는다)

## 1. 목표

- **하루의 단위를 개수에서 시간으로.** `createCallQueue`의 고정 5통을 시계(9:00→마감)로 교체. 콜 총량은 진료 소요시간에 따라 창발한다.
- **벽을 능동 거절에서 전문의 점유로.** "받을 수 있는데 안 받는다"(능동 악행 버튼)를 제거. 대신 배후과 의사가 돈 되는 예약진료로 점유돼 응급에 자유롭지 않으면 못 받는다 — 현실 뺑뺑이 1위 원인(전문의 부재 41.9% > 병상 15%)과 정합.
- **결정론 유지.** 소요시간은 `Math.random`이 아니라 **seed 결정론 난수** — 같은 (주,날)은 항상 같은 전개(테스트·재현 가능, RNG-0 헌법 준수).
- **판정·경제 핵심은 계승.** adjudicateTransfer(배후과 판정)·CALL_ECONOMICS(적자 응급)·검사 흑자·boarding 사슬·야간 당직 규칙은 그대로 살리되 **자원을 병상→전문의 시간**으로 옮긴다.

## 2. 배경 — 왜 바꾸나

지금 하루는 `ReceivingState`(콜 5통 큐)이고, 플레이어는 콜마다 수용/거절을 능동으로 누른다. 두 문제:

1. **능동 거절이 이상하다.** 자리가 있는데 STEMI를 거절하는 건 만화적 악행 버튼처럼 읽힌다. 현실 병원의 거절은 능동이 아니라 구조적이다("전문의가 지금 딴 걸 하느라 없다"). 사용자 지적.
2. **벽의 정체가 병상으로 형상화돼 있다.** NO_BED(자리 0)가 하루 총량 벽인데, 현실 지배 병목은 병상(15%)이 아니라 **배후 전문의 부재/점유(41.9%)**다([medical-system-grounding.md](../../research/medical-system-grounding.md)가 이미 지적). 그 부재의 메커니즘은 [physician-workflow-and-backup-occupancy.md](../../research/physician-workflow-and-backup-occupancy.md): **평일 주간 배후 전문의가 외래·예정수술로 점유돼 응급에 못 내려온다.**

시간 모델이 이 둘을 한 번에 푼다: 진료 소요시간이 의사를 점유하면, "돈 되는 예약으로 배후과를 채운 선택"이 나중 응급 앞의 벽으로 돌아온다. 플레이어는 거절을 누른 적이 없다 — 구조가 이미 다 썼을 뿐이다. 이게 "문제는 개인이 아니라 시스템"을 더 날카롭게 만든다.

## 3. 확정 설계 결정 (브레인스토밍 6갈래)

| 갈림 | 결정 |
|---|---|
| 시간의 역할 | **자리 벽을 시간화** — 추상 카운터(beds)를 "지금 자유로운 전문의"로 구체화 |
| 흐름 이상함 | 능동 거절 제거 — 응급 거절은 **점유의 구조적 귀결**로 |
| 플레이어 능동 결정 | **미용/예약진료를 받을지**(의사 점유 커밋) 하나로 수렴 |
| 응급 수용 | 자유 의사 있으면 **자동 수용**, 없으면 벽 (거절 버튼 없음) |
| 벽의 정체 | **배후과 전문의의 시간**(병상 아님) |
| 랜덤 | **seed 결정론 난수**(RNG-0 헌법 유지) |

## 4. 시간 모델

### 4.1 시계와 이벤트

- 하루 = 분 단위 타임라인 `[0, DAY_LENGTH_MIN]`. 표시는 09:00에서 시작(0분=09:00). **주간 창** `[0, NIGHT_START_MIN)` + **야간 창** `[NIGHT_START_MIN, DAY_LENGTH_MIN]`.
  - 튜닝값(스펙 기본): `DAY_LENGTH_MIN`·`NIGHT_START_MIN`은 구현에서 확정(예: 주간 9시간+야간 꼬리). **값은 임의 조정 가능**, 논지는 값에 안 걸린다.
- **이산 이벤트 시뮬레이션.** 실시간으로 시계가 흐르지 않고 **"다음 이벤트 시각으로 점프"**한다(사용자 요구: 실시간 카운트다운의 피로감 방지). 이벤트 2종:
  - **도착(arrival)**: 콜 한 통이 그 시각에 들어온다.
  - **완료(completion)**: 의사가 진료를 끝내 그 시각에 자유로워진다.
- 표시 시계 = 지금 처리 중인 이벤트의 시각.

### 4.2 seed 결정론 소요시간

- 종목별 `[minMin, maxMin]`(예: 보톡스 30~60, 스텐트 90~150, STEMI 60~120 — 구현 확정, 각색). 소요시간 = **순수 해시 seed**를 `[min, max]`에 매핑.
- seed = 안정 입력에서만 파생: `(week, day, arrivalIndex)` → 정수 해시(splitmix32류, `Math.random`·`Date.now` 금지). **같은 (주,날) → 항상 같은 소요시간**(같은 큐 = 같은 전개, 콜 id 체계·테스트 재현성 유지).
- 도착 시각도 같은 방식으로 seed 파생(주간 창에 결정론 분포). → 하루 전체가 결정론.

## 5. 의사 = 시간 에이전트

[doctor-roster](2026-07-20-doctor-roster-visualization-design.md)의 유닛(roster)이 곧 시뮬레이션 행위자가 된다. 유닛을 "외래용/응급용"으로 **나누지 않는다** — 같은 유닛, 점유 상태 하나. (근거: 현실도 배후과 전문의가 외래로 점유돼 있다가 자유로울 때만 응급 배후로 불려온다.)

- 상태: 각 의사 `busyUntil: number`(자유=현재 시각 이상까지 안 잡힘). 시각 t에서 **자유** ⟺ `busyUntil ≤ t`.
- 진료 배정: 자유 의사에게 배정하면 `busyUntil = t + 소요시간`. 완료 이벤트에서 자유로 돌아온다.
- **피로(PR #79)는 이제 실제 점유가 굴린다** — 하루 누적 점유 시간·야간 점유가 `stepFatigue`의 입력이 된다(현행 케이스 카운트에서 시간 기반으로 자연 승격).

## 6. 두 흐름과 벽

### 6.1 예약·선택진료 (elective) — 유일한 능동 결정

- 도착 시 플레이어가 **받을지 정한다**. 받으면 그 과 자유 의사 하나를 [소요시간] 점유하고 **흑자**를 쌓는다. 스킵하면 의사는 자유로 남는다(응급 대비).
- 자유 의사가 없으면 못 받는다(표시만, 응급 아니라 신문 없음).
- 두 종류:
  - **미용·검진**(AESTHETICS·CHECKUP): 흑자, **응급과 경쟁 안 함**(별도 과). 미용 의원이 응급 안 받는 현실과 정합.
  - **배후과 예약진료**(순환기 등): 흑자, **자기 과 응급과 같은 의사를 두고 경쟁**한다 → 여기서 비극이 난다. 근거: 심장내과 시술 과 단위 흑자·검체 160%([fee-schedule-and-subsidies.md](../../research/fee-schedule-and-subsidies.md)) — 배후과가 응급에 못 가는 이유가 *돈 안 되는 일*이 아니라 *돈 되는 예정수술·시술*이다.

### 6.2 응급 전원 (emergency) — 자동 수용·벽

- seed 시각에 도착. **판정 로직은 그대로**(adjudicateTransfer: requiredSpecialty 배후과 요구). 단 자원 게이트가 병상→전문의 점유로 바뀐다:
  - 그 과 **자유 의사가 있으면 자동 수용**(점유 `busyUntil = t + 소요`), 없으면 **벽**(turnedAway → 신문).
  - GENERAL_EMERGENCY: 기존 `hasErOnCall` + 외과(GENERAL_SURGERY) 자유 의사(handlingDept 매핑 계승). NO_BED는 제거.
  - 야간(t ≥ NIGHT_START): 기존 **당직 2인 규칙**(`roundTheClockBackup`) 유지 — 야간엔 외래가 없어 점유가 아니라 "당직이 서 있나"가 게이트. 근거: 야간 배후는 별도 온콜 인력이 서야 성립.
- 플레이어는 거절을 안 누른다 — 결과(수용 by Dr.X / 벽 because 전원 점유중)를 **본다**.

### 6.3 새 거절 사유

- `NO_BED` 제거. 대신 **`NO_FREE_SPECIALIST`**(그 과 의사가 다 진료 중) 신설 — RejectionReason에 추가. `NO_BACKUP_CARE`(과 자체가 없음)·`NO_NIGHT_BACKUP`(야간 당직 공백)와 **다른 사유**다: 과도 있고 당직도 서는데 **지금 다 바쁘다**. 이게 "평일 배후진료 공백"의 정확한 형상화다.

## 7. 기존 시스템 통합

| 기둥 | 처리 | 근거·주의 |
|---|---|---|
| **beds / bedsFree / FIXED_BEDS / NO_BED / overcrowded** | **제거.** 벽은 전문의 점유. `MAX_DOCTORS_PER_DEPT`(=beds 파생)도 재정의 필요 → 동시 처리량·당직 로테이션 기준으로. | 자리라는 추상 카운터가 전문의 시간으로 구체화돼 역할 중복 |
| **workup(검사 붙이기)** | **응급 붙이기 액션 제거, 역할 이전.** 응급이 자동수용이라 붙이기 순간이 사라진다. 흑자+boarding 역할은 **예약진료 흐름(§6.1 배후과 예약)**이 계승 — "검사 예약"이 흑자이자 boarding원. ⚠️ **사용자 검토 포인트**: 이는 현 withWorkup의 "장부에서 과잉진료가 창발" 순간을 예약진료로 옮긴다. | "덮는 게 보인다"는 두 줄 장부(적자 응급 + 흑자 예약) 그대로. 오히려 덮는 흑자와 벽의 원인이 **같은 활동**(예약 점유)이 돼 논지가 더 선명 |
| **boarding(내일 자리 점유)** | **자원만 병상→의사.** boarding원 예약진료(검사)는 `busyUntil > DAY_LENGTH`로 그 의사를 **내일까지 점유** → 내일 그 과 수용력이 준다. | 저수가→흑자→대가는 내일 수용력 잠식→신문, 이 사슬 보존 |
| **dayProgress** | `index/큐길이` → **`clock / DAY_LENGTH`**. accruedSegments가 이걸로 부문 손익을 누적. | 콜 수가 창발하니 시간이 진행률 기준. 7일 완주 시 주간 전액 도달 등식(§4-1 정합) 유지 |
| **DEPARTMENTS 구조 손익(-12 등 24h 대기 고정비)** | **유지.** 배후과 net = 대기 고정비(−) + 예약 흑자(+, 플레이 시) − 응급 적자(−). | 대기비와 시술 수익은 다른 층(이중계상 아님). ⚠️ 밸런스는 §9 리스크 |
| **CALL_ECONOMICS(적자 응급)** | 그대로. | 부호·방향 근거 유지 |
| **야간 당직(roundTheClockBackup, 2인)** | 그대로(§6.2). | 근거 있음 |
| **아침 신문 / turnedAway / lawsuit** | 그대로 — 벽 사유가 NO_BED→NO_FREE_SPECIALIST로 바뀔 뿐, "못 받은 필수 응급 = 신문" 집합은 동일. | |

## 8. 결정론 · 테스트 (TDD)

- **순수·결정론 유지가 대전제.** seed 해시는 순수 함수, 같은 입력=같은 출력. 하루 전체(도착 시각·소요시간·배정·벽)가 (주,날)에서 결정론 파생 → 기존처럼 테스트 가능.
- TDD 대상(구현 plan에서 태스크로): seed 소요시간(경계·재현), 도착 스케줄(결정론·주간/야간 분포), 점유 배정(자유 의사 선택·벽 판정), 야간 당직 게이트, boarding(내일 이월), dayProgress(시간 기반), 경제 누적(elective 흑자/응급 적자 분리).
- 게이트: `vitest` 전량 green + `tsc --noEmit` 0([vitest-no-typecheck-tsc-gate] — esbuild가 타입 안 봄).

## 9. 리스크 · 미해결

- **경제 밸런스(I8: |순이익| ≤ 4×예산).** 배후과에 흑자원(예약)을 주면 스케일이 깨질 수 있다(미용 10명이 I8 깬 전례 — setup.ts:22). **부호·방향만 근거, 금액 각색**하고, 불변식은 **브라우저 7일 완주로만** 잡힌다(receiving.ts:35). plan에 7일 완주 검증 태스크 필수.
- **thesis 이동**: "배후과는 무조건 적자"에서 "배후과는 흑자활동(시술)과 응급대기 사이에서 흑자를 좇으면 응급이 막힌다"로. 이는 리서치가 지지하는 **더 정확한** 명제지만, 기존 essential-care-economics 서술과 톤 정합 필요. ⚠️ 사용자 검토 포인트.
- **UX 피로**: elective 결정이 너무 잦으면 지친다. 하루 도착 밀도를 튜닝(≈현행 5통 리듬)으로 잡는다.
- **정답-퍼즐 방지**: "응급 대비로 비워두기"가 항상 정답이 되면 퍼즐이다. 방어: 응급은 적자라 무조건 비워두기가 이익도 아니고, 도착 시각이 seed라 언제 올지 모른다(외워서 못 풂). 오히려 불확실성이 퍼즐을 막는다.
- **MAX_DOCTORS_PER_DEPT 재정의**: 지금 beds 파생인데 beds가 사라진다. 동시 처리량·당직 기준으로 새로 잡는다(현재 상한 3의 의미가 시간 모델에선 "동시에 몇 진료"로 자연스러워짐).

## 10. UI (개략 — 렌더 세부는 구현)

- RECEIVING 화면: 상단 **시계**(현재 시각) + 의사 **타임라인/로스터 스트립**(누가 지금 무엇으로 점유, 누가 자유). PR #79 로스터를 시간 축으로 확장.
- elective 도착 = 결정 카드(받기/보내기). emergency 도착 = 결과 배너(수용 by Dr.X / 벽: 전원 점유중 → 뺑뺑이).
- **해석 카피 0**(show-don't-tell): 시각·점유·벽만 보여주고 "당신 탓" 같은 문장은 없다.

## 11. 0-침습 경계 (무엇을 안 건드리나)

- adjudicateTransfer의 **배후과 판정 로직**(requiredSpecialty)은 무변경 — 자원 게이트(병상→전문의 점유)만 교체.
- world·setup 위저드·경제 부호 근거·에필로그 장부·아침 신문 렌더는 유지. 소송 비용은 결말에서만(1막 명랑 숫자 원칙).

## 12. 스코프 밖 (지금 안 함)

- 종목별 소요시간·도착 밀도·하루 길이 **최종 수치**(구현에서 튜닝).
- 피로가 판정에 개입(승격 문 — 여전히 표시/시간에만).
- 흉부외과 라우팅(받는 콜 종류 없음 — 현행 유지, 정직한 0).
- 실시간 애니메이션 타임라인(정적 이벤트 점프로 충분).
